# breadmin-jobman v0.3.0 — Signals Pipeline

**Theme:** Schedule Reddit ingestion and breadwinner options/crypto state as regular
signals jobs.

---

## Overview

Two integration jobs that expand what moot councils have access to:

1. **Reddit signals job**: the `breadmin-reddit` package already fetches and archives
   aggregated discourse data via `RedditSync`. What doesn't exist is this being registered
   as a scheduled breadmin job that writes output to jobman's signals store (not directly
   to moot's `DiscourseLibrary`), with proper error handling, alerting, and staleness
   tracking. This milestone wires it. The signals store is jobman-owned SQLite; moot reads
   it via the path in `JOBMAN_SIGNALS_DB`. No direct moot imports in job handlers.

2. **Breadwinner portfolio snapshot job**: the `breadwinner-state-snapshot` job from
   v0.2.0-verdict-bridge delivers a full snapshot at deliberation time. This milestone
   adds a lighter-weight scheduled job that maintains a continuously-fresh snapshot in
   the portfolio state store, so the financial council always has recent context without
   waiting for the snapshot job to run on demand.

Both jobs are signal producers. The consumers (moot councils) are unchanged; they read
from the same stores already defined in v0.2.0 and moot v0.7.0.

**Prerequisite:** v0.2.0-verdict-bridge must complete (portfolio state store schema
defined); moot v0.7.0-open-field must be in progress (signals store consumption layer).

## Success Criteria

- [ ] A `reddit-sync` job is registered as a breadmin schedule (configurable interval,
  default: every 6 hours) and calls `RedditSync.sync_all()`; output is written to the
  signals store (jobman's SQLite, path in `JOBMAN_SIGNALS_DB`) with a `fetched_at`
  timestamp; no direct write to moot's `DiscourseLibrary`
- [ ] Staleness is tracked: when the most recent `reddit-sync` result is older than 24
  hours, the job writes a staleness flag that moot's context layer can read
- [ ] `reddit-sync` failures (Reddit API unavailable, auth failure, rate limit) are
  handled per the failure modes below and do not block moot deliberations
- [ ] A `breadwinner-portfolio-refresh` job runs on a schedule (every 30 minutes during
  market hours, configurable) and updates the portfolio state store with fresh
  options/crypto state — independent of deliberation triggers
- [ ] The portfolio state store always has an entry fresher than the configured staleness
  threshold during market hours; the financial council's context injection layer can
  detect staleness
- [ ] `make test-reddit && make test-core && make lint` pass on mainline

## Scope

**Included:**

- `reddit-sync` job handler: thin wrapper around `RedditSync.sync_all()` with breadmin
  error handling and staleness tracking; writes to the signals store only — no direct
  moot `DiscourseLibrary` write; `_write_to_moot()` in `breadmin_reddit/sync.py` is
  deleted as part of this milestone
- Schedule config for `reddit-sync` (default: every 6 hours, active hours)
- Staleness flag write: when `sync_all()` succeeds, write timestamp to signals store;
  moot reads this to check freshness
- `breadwinner-portfolio-refresh` job handler: lightweight version of the v0.2.0
  `breadwinner-state-snapshot` — runs on a tighter cadence without triggering deliberation
- Schedule config for `breadwinner-portfolio-refresh` (default: every 30 min, market hours)
- Alerting: Telegram notification (if configured) when `reddit-sync` fails 3 consecutive
  times or `breadwinner-portfolio-refresh` is stale for > 2 hours during market hours

**Explicitly excluded:**

- Reddit fetching, parsing, or sentiment logic (breadmin-reddit handles it)
- Portfolio state store schema changes (defined in v0.2.0)
- Moot consumption layer changes (moot v0.7.0 scope)
- Real-time Reddit streaming

## Failure Modes

**reddit-sync:**
- Reddit API unavailable → log warning; mark signals store as stale; retry on next
  scheduled run; do not alert until 3 consecutive failures
- Reddit API auth failure → log error; disable Reddit sync for the session; alert
  operator via Telegram (if configured); do not retry until credentials are fixed
- Reddit API rate limit → `RedditSync` handles backoff internally; job logs rate-limit
  event and completes normally
- Subreddit not found or private → `RedditSync` skips it; job completes with partial
  results; log warning

**breadwinner-portfolio-refresh:**
- Breadwinner CLI/API unavailable → log warning; skip cycle; do not alert until
  stale threshold exceeded
- Portfolio state store write fails → log error; retry once; alert operator if failing
  for > 2 consecutive cycles during market hours
- Market closed → job skips silently if outside configured market hours; no alert

## Interface Contracts

**Signals store freshness record (written by jobs, read by moot):**

```json
{
  "source": "reddit | breadwinner",
  "last_updated": "<iso8601>",
  "is_stale": false,
  "stale_threshold_hours": 24
}
```

**Schedule config (YAML):**

```yaml
reddit-sync:
  schedule: "0 */6 * * *"          # Every 6 hours
  active_hours_only: false
  handler: breadmin_reddit.cli:reddit
  args: ["sync", "--json-only"]
  max_retries: 2
  alert_after_failures: 3

breadwinner-portfolio-refresh:
  schedule: "*/30 9-16 * * 1-5"    # Every 30 min, 9am-4pm weekdays
  active_hours_only: true
  handler: breadwinner.cli:snapshot
  args: ["--format=jobman"]
  max_retries: 1
```

---

## Constraints

- `reddit-sync` failure must never block a moot deliberation — staleness warning only
- No direct imports from moot in job handlers — all writes via the contract schemas
- Jobman owns the signals store; moot reads it via `JOBMAN_SIGNALS_DB` env var (SQLite
  path); moot never writes to the signals store
- Schedule configs are file-based and gitignored (same as all breadmin configs)
- All new code is type-annotated and passes `ruff check`
