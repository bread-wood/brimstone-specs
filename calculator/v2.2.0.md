# Calculator CLI v2.2.x тАФ Simplify

**Theme:** The expression knows when it can be shorter.

## Overview

Adds symbolic simplification. `calc simplify 'sin^2(x) + cos^2(x)'` returns
`1`; `calc simplify '(x^2 - 4) / (x - 2)'` returns `x + 2`; `calc simplify
'2*x + 3*x'` returns `5*x`. Simplification cleans up the output of symbolic
differentiation (which may produce `3*x^1` instead of `3*x`) and makes the
calculator useful for algebraic manipulation. The final version in the v2.x
symbolic layer, and the one that makes all prior symbolic features more
practically useful.

## Scope

**Included:**

- `calc simplify '<expression>'` subcommand; returns simplified expression string
- Algebraic simplification rules:
  - Arithmetic: `x + 0 тЖТ x`, `x * 1 тЖТ x`, `x * 0 тЖТ 0`, `x^1 тЖТ x`, `x^0 тЖТ 1`
  - Like terms: `2*x + 3*x тЖТ 5*x`
  - Factoring: `(x^2 - 4) / (x - 2) тЖТ x + 2`
  - Trigonometric identities: `sin^2(x) + cos^2(x) тЖТ 1`
  - Exponential/log inverses: `exp(log(x)) тЖТ x`, `log(exp(x)) тЖТ x`
  - Power rules: `x^a * x^b тЖТ x^(a+b)`
- Applied automatically to derivative output: `calc diff` output is simplified
  before printing
- `--latex` flag: simplified output as LaTeX

---

## Failure Modes

**Simplification:**
- Expression is already in simplest form тЖТ return it unchanged; not an error
- Simplification would require assumptions about variable sign or domain
  (e.g. `sqrt(x^2)` тЖТ `x` requires `x тЙе 0`) тЖТ return unchanged; emit warning
  `warning: could not simplify <expr> without domain assumptions`

---

## Interface Contracts

**CLI:**

```
calc simplify '<expression>' [--latex]
```

| Stream | Content |
|--------|---------|
| stdout | Simplified expression string (or LaTeX) |
| stderr | Warnings for expressions that could not be simplified |
| exit 0 | Always succeeds (worst case: returns input unchanged) |

---

## Constraints

- Depends on v2.0.x for the CAS representation; simplification operates on
  the same AST used by symbolic differentiation
- Simplification is a best-effort operation; it is not guaranteed to produce
  the globally minimal expression
- `calc diff` output is automatically simplified from this version forward;
  the simplification step is transparent to the user
- Simplification rules are implemented as a rewrite rule system; adding a new
  rule requires adding one entry, not modifying the simplifier's core loop

---

## Key Unknowns

1. **[P1]** Is a rewrite-rule simplifier тАФ applied repeatedly until no rule
   fires тАФ sufficient to handle the algebraic identities needed in practice
   (trig identities, factoring, inverse function cancellation), or do some
   simplifications require a canonical form (e.g. polynomial normal form)
   that a rule system alone cannot reliably reach? The answer determines
   whether the simplifier is a rule engine or requires a partial CAS with
   canonical form computation.

---

## Non-Goals

- Proof of simplification correctness
- Simplification with assumptions (e.g. `x > 0`)
- Polynomial factoring over integers (requires a full CAS)
- Simplification of differential equations
- Windows support
