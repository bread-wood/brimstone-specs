# Calculator CLI v0.7.x — Interactive

**Theme:** The plot talks back. Pan, zoom, inspect.

## Overview

Adds an interactive window mode that displays the plot live and responds to
mouse and keyboard input. Expressions are still supplied on the command line;
there is no in-window editing in this version. The user can pan the viewport,
zoom in and out, and move a cursor along curves to read off coordinates. This
is the first version that requires a windowing system and the version that
validates the rendering pipeline established in v0.5.x can drive a live display.

---

## Success Criteria

- [ ] `calc plot 'sin(x)' --interactive` opens a window displaying the curve and
  exits 0 after the window is closed
- [ ] `calc plot 'sin(x)' 'cos(x)' --interactive` opens a window with both curves
  and the legend visible
- [ ] Scrolling up in the window zooms in; scrolling down zooms out; the curve
  redraws without visible lag (verified in automated UI tests via synthetic events)
- [ ] Click-and-drag pans the viewport; the curve follows the drag
- [ ] Moving the mouse along a curve snaps the cursor to the nearest curve and
  displays `x = <value>, y = <value>` in the status bar
- [ ] Pressing `R` resets the viewport to the initial domain and y-range
- [ ] Pressing `S` saves the current viewport as a PNG to `./plot.png` (or
  the path from `--output`) and displays a brief confirmation in the status bar
- [ ] Pressing `Q` or closing the window exits 0
- [ ] `calc plot 'sin(x)' --interactive --screenshot out.png` renders the initial
  view to `out.png` without opening a window and exits 0 — for CI use
- [ ] `calc plot 'sin(x)' --interactive` on a system with no display available
  prints `error: no display available` to stderr and exits 1
- [ ] `make test` passes clean on macOS and Linux; interactive tests use
  `--screenshot` mode

---

## Scope

**Included:**

- `--interactive` flag on `calc plot`
- Window displays the same output as the file renderer for the same expression
  and domain
- Pan: click-and-drag moves the viewport
- Zoom: scroll wheel zooms centered on the cursor position
- Cursor readout: mouse position snapped to nearest curve; x/y displayed in
  status bar
- Viewport reset: `R` key
- Save current view: `S` key writes to the `--output` path (default `./plot.png`)
- Quit: `Q` key or window close button
- `--screenshot PATH` flag: renders the initial view to file and exits without
  opening a window; enables CI testing of visual output
- All multi-curve and parametric features from v0.6.x work in interactive mode

---

## Failure Modes

**Display:**
- No display available (headless environment without `--screenshot`) →
  `error: no display available` to stderr, exit 1
- Window creation fails → `error: could not create window: <reason>` to stderr, exit 1

**Save:**
- `S` key pressed but output path not writable → displays `error: cannot save: <reason>`
  in the status bar; does not exit

**Input:**
- `--interactive` combined with `--output` without `S` key or `--screenshot` →
  `--output` is silently reserved for `S` key saves; no conflict

## Interface Contracts

**CLI:**

```
calc plot '<expr>' [additional exprs...] --interactive [--screenshot PATH] [all v0.6.x flags]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--interactive` | off | Open a live window instead of writing a file |
| `--screenshot PATH` | — | Render initial view to file and exit; no window opened |

**Keyboard bindings:**

| Key | Action |
|-----|--------|
| `R` | Reset viewport to initial domain and range |
| `S` | Save current viewport to `--output` path |
| `Q` | Quit |
| `Esc` | Quit |

**Mouse bindings:**

| Gesture | Action |
|---------|--------|
| Scroll up | Zoom in, centered on cursor |
| Scroll down | Zoom out, centered on cursor |
| Click + drag | Pan viewport |
| Mouse move | Update cursor readout in status bar |

| Stream | Content |
|--------|---------|
| stdout | Empty |
| stderr | `error: <description>` on fatal startup failure |
| exit 0 | Window closed normally or `--screenshot` completed |
| exit 1 | Fatal error (no display, window creation failure) |

## Constraints

- All v0.5.x and v0.6.x constraints remain in force for file output
- The interactive renderer must use the same curve-data model as the file renderer;
  no duplication of the expression evaluation or sampling logic
- Redraw on pan/zoom must complete within one frame at 60 fps on commodity hardware
  for expressions that evaluate in under 1 ms per sample
- `--screenshot` mode must produce pixel-identical output to the equivalent
  `calc plot` file command for the same expression and domain
- The window and event model must extend to in-window text input for expression
  editing in v0.8.x without replacing the windowing framework

---

## Key Unknowns

1. **[P1]** Does the windowing and event framework chosen for pan/zoom/cursor
   interaction extend to text input field rendering and keyboard event routing
   for in-window expression editing in v0.8.x — or does adding a text field
   require replacing the UI layer with a toolkit that supports retained-mode
   widgets? The answer determines whether v0.7.x establishes the permanent
   UI framework or is a stepping stone that must be superseded.

---

## Non-Goals

- Expression entry or editing in the window — v0.8.x
- Saving session state (expressions, viewport) — v1.0.x
- Implicit curves and inequalities — v0.9.x
- Polar or parametric axes in the interactive view (parametric curves display
  correctly but axes remain Cartesian)
- Windows support
