# Calculator CLI v0.8.x — Editor

**Theme:** Type in the window. The calculator becomes a graphing calculator.

## Overview

Adds an in-window expression editor so users can add, edit, and remove curves
without leaving the window or returning to the command line. The window opens
with any expressions passed on the command line pre-populated; from there the
user types to add more, edits existing ones, and the plot updates live after
each keystroke. This is the version where the product becomes a graphing
calculator in the Desmos sense — a self-contained tool that does not require
the terminal for day-to-day use.

---

## Success Criteria

- [ ] `calc plot --interactive` opens a window with an empty expression list and
  an active input field; exits 0 when the window is closed
- [ ] Typing `sin(x)` in the input field and pressing Enter adds a curve labeled
  `sin(x)` to the plot and clears the field for a new expression
- [ ] `calc plot 'sin(x)' --interactive` opens with `sin(x)` pre-populated and
  its curve already visible
- [ ] Editing an existing expression (clicking its label, modifying text) updates
  the curve live on each keystroke; the old curve is replaced as soon as the
  updated expression is valid
- [ ] An expression with a syntax error is highlighted in red in the expression
  list; the plot retains the last valid version of that curve
- [ ] Deleting an expression from the list removes its curve from the plot
- [ ] `calc plot --interactive --screenshot out.png` with no expressions produces
  an image of an empty plot and exits 0
- [ ] `calc plot --interactive --screenshot out.png` with `'sin(x)'` pre-populated
  produces an image with the curve visible and exits 0
- [ ] All v0.7.x keyboard and mouse interactions (pan, zoom, cursor readout, R, S, Q)
  still work in the editor window
- [ ] `make test` passes clean on macOS and Linux using `--screenshot` mode

---

## Scope

**Included:**

- Expression list panel: sidebar or bottom panel listing all active expressions,
  each with its label color matching its curve
- New expression input: a text field at the end of the expression list; Enter
  commits the expression and opens a new blank field
- Inline editing: clicking an existing expression label makes it editable in place
- Live rerender: plot updates after each keystroke; if the expression is invalid,
  the last valid curve is retained and the field is highlighted in red
- Expression deletion: a remove button or keyboard shortcut per expression
- Expressions from the command line are pre-populated on open
- All v0.7.x pan, zoom, cursor readout, R, S, and Q interactions unchanged

---

## Failure Modes

**Expression editing:**
- Invalid expression during live edit → field highlighted red; last valid curve
  retained; no error dialog or exit
- Empty expression committed (Enter on blank field) → ignored; no blank curve added

**Window:**
- All v0.7.x window failure modes (no display, window creation failure) apply unchanged

---

## Interface Contracts

**CLI:**

```
calc plot ['<expr>' ...] --interactive [--screenshot PATH] [all v0.7.x flags]
```

No new CLI flags in this version; all interaction is in-window.

**In-window expression panel:**

| Action | Result |
|--------|--------|
| Type in new-expression field + Enter | Adds curve; opens new blank field |
| Click existing expression label | Opens it for inline editing |
| Edit expression text | Live rerender; red highlight on invalid input |
| Press Escape while editing | Reverts to last committed value |
| Click remove button on expression | Removes curve and expression row |

| Stream | Content |
|--------|---------|
| stdout | Empty |
| stderr | `error: <description>` on fatal startup failure |
| exit 0 | Window closed normally or `--screenshot` completed |
| exit 1 | Fatal error (no display, window creation failure) |

---

## Constraints

- All v0.7.x constraints remain in force
- Live rerender must not block the UI thread; expression evaluation runs off the
  main thread; partial results are acceptable during long evaluations
- The expression list and its associated curve state must be representable as a
  serializable structure in preparation for session persistence in v1.0.x
- The UI layout must accommodate the expression panel without reducing the plot
  area below 60% of the window width

---

## Key Unknowns

1. **[P1]** Can the in-memory expression list and curve state be serialized and
   reloaded as a persistent session in v1.0.x — specifically, do the data
   structures chosen to support live editing (mutable, event-driven) map cleanly
   to a serializable format, or does adding persistence require a separate
   representation that must be kept in sync with the editor state? The answer
   determines whether v0.8.x data structures are the foundation for sessions or
   a layer that must be adapted.

---

## Non-Goals

- Session save and load — v1.0.x
- Implicit curves and inequalities — v0.9.x (these can be added to the expression
  list in v0.9.x using the same editor)
- Variable sliders (e.g. Desmos-style `a` slider) — future version
- Collaboration or shared sessions
- Windows support
