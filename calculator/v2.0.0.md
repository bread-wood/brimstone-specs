# Calculator CLI v2.0.x — Symbolic Differentiation

**Theme:** The derivative as an expression, not a number.

## Overview

Adds symbolic differentiation. `calc diff 'x^3 + 2*x' x` returns `3*x^2 + 2`
as an expression, not a numerical approximation. The result is usable anywhere
an expression is: it can be evaluated, plotted, differentiated again, or
exported to LaTeX. The graphing window gains a tangent line overlay: for any
curve, `T` toggles a tangent line that follows the cursor. This version
introduces the CAS (computer algebra system) representation layer that symbolic
simplification and equation solving will build on in v2.1.x and v2.2.x.

## Success Criteria

- [ ] `calc diff 'x^3 + 2*x' x` prints `3*x^2 + 2` to stdout and exits 0
- [ ] `calc diff 'sin(x)' x` prints `cos(x)` to stdout and exits 0
- [ ] `calc diff 'exp(x)' x` prints `exp(x)` to stdout and exits 0
- [ ] `calc diff 'x^2 * sin(x)' x` prints `2*x*sin(x) + x^2*cos(x)` (product rule)
  and exits 0
- [ ] `calc diff 'sin(x^2)' x` prints `cos(x^2) * 2*x` (chain rule) and exits 0
- [ ] `calc diff 'x^2 + y^2' x` prints `2*x` (partial; treats `y` as constant)
  and exits 0
- [ ] `calc diff 'x^2 + y^2' y` prints `2*y` and exits 0
- [ ] `calc diff 'sin(x)' x x` (second derivative) prints `-sin(x)` and exits 0
- [ ] `calc 'def f(x) = x^3; diff(f(x), x)'` prints `3*x^2` and exits 0
- [ ] In the graphing window, pressing `T` toggles a tangent line at the cursor
  position for the nearest curve
- [ ] `calc diff 'abs(x)' x` at `x = 0` prints `error: not differentiable at x=0`
  and exits 1
- [ ] `make test` passes clean

## Scope

**Included:**

- `calc diff '<expression>' <variable>` subcommand; returns derivative as expression string
- Repeated differentiation: `calc diff '<expr>' x x` for second derivative;
  `calc diff '<expr>' x x x` for third
- Differentiation rules: constant, power, sum, product, quotient, chain, and
  known derivatives for all built-in functions
- Partial differentiation: differentiate with respect to any named variable;
  other names treated as constants
- User-defined function differentiation: `diff(f(x), x)` applies the chain rule
- LaTeX output: `calc diff '<expr>' x --latex` returns the derivative as LaTeX
- Tangent line overlay in interactive graphing: `T` key toggles at cursor position

---

## Failure Modes

**Differentiability:**
- Expression not differentiable at a specific point (e.g. `abs(x)` at `x = 0`) →
  for symbolic output, return the symbolic derivative where it exists;
  for evaluation, `error: not differentiable at <variable>=<value>` to stderr, exit 1
- Expression contains a construct with no symbolic derivative rule →
  `error: cannot differentiate: <construct>` to stderr, exit 1

---

## Interface Contracts

**CLI:**

```
calc diff '<expression>' <variable> [<variable> ...] [--latex]
```

| Argument | Purpose |
|----------|---------|
| `<expression>` | Expression to differentiate |
| `<variable>` | Variable to differentiate with respect to; repeat for higher orders |
| `--latex` | Output derivative as LaTeX string |

**Differentiation rules implemented:**

| Form | Derivative |
|------|-----------|
| constant | 0 |
| `x^n` | `n*x^(n-1)` |
| `f + g` | `diff(f) + diff(g)` |
| `f * g` | `f*diff(g) + diff(f)*g` |
| `f(g(x))` | `diff(f)(g(x)) * diff(g)` |
| `sin(x)` | `cos(x)` |
| `cos(x)` | `-sin(x)` |
| `exp(x)` | `exp(x)` |
| `log(x)` | `1/x` |
| `sqrt(x)` | `1/(2*sqrt(x))` |

| Stream | Content |
|--------|---------|
| stdout | Derivative as expression string (or LaTeX with `--latex`) |
| stderr | `error: <description>` on failure |
| exit 0 | Success |
| exit 1 | Any error |

---

## Constraints

- All prior constraints remain in force
- Symbolic differentiation operates on the AST; the result is a new AST that
  is printed as an expression string using the same printer as the REPL
- The output expression is not automatically simplified; `3*x^(2-1)` may appear
  as `3*x^1` rather than `3*x` until v2.2.x (simplification)
- The CAS representation introduced in this version must extend to symbolic
  integration and equation solving in v2.1.x and v2.2.x without a new
  representation layer

---

## Key Unknowns

1. **[P1]** Does the existing AST — used for numerical evaluation — serve as a
   sufficient representation for symbolic manipulation (differentiation, later
   simplification), or does symbolic computation require a distinct CAS
   representation that must be kept in sync with the evaluator AST? The answer
   determines whether the v0.1.x AST is the permanent internal representation
   or whether v2.0.x introduces a second representation layer.

---

## Non-Goals

- Automatic simplification of derivative output — v2.2.x
- Symbolic integration — v2.1.x
- Implicit differentiation
- Differential equations
- Windows support
