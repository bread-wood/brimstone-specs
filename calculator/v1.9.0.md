# Calculator CLI v1.9.x — Data

**Theme:** Real numbers from the real world. CSV in, curve out.

## Overview

Adds data import and scatter plotting. A user can load a CSV file, plot the
raw data as a scatter plot, overlay a function curve, and fit a regression
line — all in one command. The calculator shifts from pure mathematics to
light data analysis: model a dataset, visualise the fit, compute residuals.
Targets the researcher or engineer who has data and a candidate model and wants
to see how well they match without leaving the terminal.

---

## Success Criteria

- [ ] `calc plot --data data.csv` renders a scatter plot of the data and exits 0
- [ ] `calc plot 'sin(x)' --data data.csv` overlays the function curve on the
  scatter plot with both visible
- [ ] `calc plot --data data.csv --fit linear` adds a linear regression line
  and annotates the plot with slope, intercept, and R²
- [ ] `calc plot --data data.csv --fit poly:2` adds a degree-2 polynomial fit
- [ ] `calc stats --data data.csv` prints descriptive statistics to stdout
  (count, mean, std, min, max, median for each numeric column) and exits 0
- [ ] `calc plot --data data.csv` where `data.csv` has columns other than `x,y`
  accepts `--xcol` and `--ycol` to select columns by name or index
- [ ] `calc plot --data missing.csv` prints `error: cannot read data file: <reason>`
  to stderr and exits 1
- [ ] `calc plot --data data.csv --fit linear --screenshot out.png` renders to
  file and exits 0
- [ ] `make test` passes clean with bundled test fixture CSVs

---

## Scope

**Included:**

- `--data PATH` flag on `calc plot`: load a CSV and render as scatter plot
- CSV format: first row optionally a header; at least two numeric columns
- Column selection: `--xcol` and `--ycol` accept column name or zero-based index;
  default to first two columns
- `calc stats --data PATH`: descriptive statistics per numeric column to stdout
- Regression fitting: `--fit linear`, `--fit poly:N` (degree N polynomial)
- Fit annotation: equation, R², plotted on the graph
- Residual output: `--residuals` flag writes residuals as a second plot panel
- Compatible with all existing plot flags (`--xmin`, `--xmax`, `--output`, etc.)

---

## Failure Modes

**Data loading:**
- File not found or not readable → `error: cannot read data file: <reason>` to stderr, exit 1
- File is not valid CSV → `error: invalid CSV: <reason>` to stderr, exit 1
- Selected column contains non-numeric values → `error: column <name> contains non-numeric data` to stderr, exit 1
- Fewer than 2 data points → `error: data file must contain at least 2 rows` to stderr, exit 1

**Fitting:**
- `--fit poly:N` with N ≥ number of data points → `error: polynomial degree must be less than number of data points` to stderr, exit 1
- Fit fails to converge → `error: fit did not converge` to stderr, exit 1

---

## Interface Contracts

**CLI:**

```
calc plot ['<expr>' ...] --data <path> [--xcol <name|index>] [--ycol <name|index>]
    [--fit linear|poly:N] [--residuals] [all existing plot flags]

calc stats --data <path>
```

**`calc stats` output (stdout):**

```
column: x
  count:  100
  mean:   5.02
  std:    2.87
  min:    0.10
  max:    9.97
  median: 5.15

column: y
  ...
```

**Fit annotation on plot:** equation string and R² displayed in the legend entry
for the fit curve.

| Stream | Content |
|--------|---------|
| stdout | Empty (plot) or stats table |
| stderr | `error: <description>` on failure |
| exit 0 | Success |
| exit 1 | Any error |

---

## Constraints

- All prior plotting constraints remain in force
- Data files must be local; no URL loading
- Maximum data file size: 10 MB; larger files produce `error: data file too large`
- The data point representation must extend to time-series data (date/time x-axis)
  in a future version without changing the CSV loading interface
- No external statistics library dependencies; regression implemented in the
  standard library

---

## Key Unknowns

1. **[P2]** Does the scatter plot data model — a list of (x, y) points with a
   label and color — compose with the existing curve data model cleanly enough
   to share axis scaling, legend rendering, and color palette management, or
   does scatter data require a fundamentally different rendering path that
   duplicates scene management code? The answer determines whether data overlay
   is a first-class scene element or a post-processing layer.

---

## Non-Goals

- Time-series or date-axis support
- Statistical hypothesis testing
- Clustering or classification
- Non-CSV input formats (JSON, Excel, Parquet)
- Remote data sources (URLs, databases)
- Windows support
