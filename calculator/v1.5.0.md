# Calculator CLI v1.5.x — Lists

**Theme:** Numbers in sequence. The evaluator handles collections.

## Overview

Adds list literals, ranges, and higher-order operations over lists. `[1..10]`
produces a list; `map(f, [1..10])` applies a function to each element;
`filter(x > 3, [1..5])` keeps elements matching a predicate;
`mean([1,2,3,4,5])` computes the average. Lists are the natural bridge between
the scalar calculator and statistical computation — and the mechanism that makes
`sum([1..100])` readable without the dedicated `sum(k, 1, 100, k)` syntax.

## Success Criteria

- [ ] `calc '[1, 2, 3]'` prints `[1, 2, 3]` to stdout and exits 0
- [ ] `calc '[1..5]'` prints `[1, 2, 3, 4, 5]` to stdout and exits 0

- [ ] `calc 'len([1, 2, 3])'` prints `3` to stdout and exits 0
- [ ] `calc '[1, 2, 3][1]'` prints `2` (zero-indexed) to stdout and exits 0

- [ ] `calc 'sum([1..100])'` prints `5050` and exits 0
- [ ] `calc 'mean([1, 2, 3, 4, 5])'` prints `3` and exits 0

## Scope

**Included:**

- List literal: `[a, b, c, ...]`
- Integer range: `[from..to]` (inclusive, step 1)
- Stepped range: `[from..to..step]`
- Index access: `list[n]` (zero-indexed)
- `len(list)` — number of elements
- `map(f, list)` — apply function to each element; returns list
- `filter(pred, list)` — keep elements where `pred(x)` is nonzero; returns list
- `reduce(f, init, list)` — fold list with binary function and initial value
- Statistical functions over lists: `sum`, `product`, `mean`, `min`, `max`,
  `std` (population standard deviation), `median`
- Lists may contain any scalar value (real, complex); heterogeneous lists
  not supported in this version

---

## Failure Modes

**Index:**
- Out-of-bounds index → `error: index out of bounds` to stderr, exit 1
- Non-integer index → `error: list index must be integer` to stderr, exit 1

**Statistical functions:**
- `std` or `mean` on empty list → `error: empty list` to stderr, exit 1
- `median` on empty list → `error: empty list` to stderr, exit 1

**Range:**
- Non-finite step in stepped range → `error: range step must be finite and nonzero` to stderr, exit 1
- Range that would produce more than 1,000,000 elements → `error: range too large` to stderr, exit 1

---

## Interface Contracts

**CLI:** Unchanged.

**Output format:**

- List printed as `[a, b, c]` with elements formatted using the standard scalar
  output rules (integer if whole, decimal otherwise)
- Empty list printed as `[]`

**New functions:**

| Function | Signature | Description |
|----------|-----------|-------------|
| `len` | `len(list)` | Number of elements |
| `map` | `map(f, list)` | Apply f to each element |
| `filter` | `filter(f, list)` | Keep elements where f(x) ≠ 0 |
| `reduce` | `reduce(f, init, list)` | Fold with binary f and initial value |
| `mean` | `mean(list)` | Arithmetic mean |
| `median` | `median(list)` | Median value |
| `std` | `std(list)` | Population standard deviation |
| `min` | `min(list)` | Minimum element |
| `max` | `max(list)` | Maximum element |

| Stream | Content |
|--------|---------|
| stdout | Scalar or list result |
| stderr | `error: <description>` on failure |
| exit 0 | Success |
| exit 1 | Any error |

---

## Constraints

- All prior constraints remain in force
- Lists are immutable; no in-place modification
- `map`, `filter`, and `reduce` accept only single-argument user-defined or
  built-in functions; no anonymous lambda syntax in this version
- The list type must extend to matrices (lists of lists) in a future version
  without changing the list literal or index syntax
- `sum` and `product` applied to a list are equivalent to the v1.2.x constructs
  with the same name; the two forms are interchangeable

---

## Key Unknowns

1. **[P1]** Does the list type — an ordered sequence of scalars — extend to
   matrices (lists of lists) and vector operations without a separate type
   representation, or does matrix support require a dedicated 2D type with its
   own syntax and evaluation rules? The answer determines whether lists are the
   permanent collection type or a stepping stone to a richer type system.

---

## Non-Goals

- Mutable lists or in-place operations
- List comprehension syntax (`[f(k) for k in list]`) — uses `map` instead
- String type
- Heterogeneous lists
- Sparse or infinite lists
- Windows support
