# Calculator CLI v0.6.x — Multi-Curve

**Theme:** More than one curve. Comparison and composition.

## Overview

Extends the v0.5.x renderer to plot multiple expressions on the same axes and
adds parametric mode for curves defined by paired expressions over a shared
parameter. Targets developers who want to compare functions visually or plot
paths through a plane. Each curve gets a distinct color and a legend entry.
The domain and y-range automatically encompass all curves.

---

## Success Criteria

- [ ] `calc plot 'sin(x)' 'cos(x)'` writes a PNG with two visually distinct curves
  and a legend, and exits 0
- [ ] `calc plot 'x' 'x^2' 'x^3' --xmin 0 --xmax 2` plots three curves on one set
  of axes and exits 0
- [ ] `calc plot --parametric 'sin(t)' 'cos(t)'` plots a unit circle and exits 0
- [ ] `calc plot --parametric 'sin(t)' 'cos(t)' --tmin 0 --tmax 3.14159` plots a
  semicircle and exits 0
- [ ] `calc plot --parametric 'sin(t)' 'cos(t)' 'sin(2*t)' 'cos(3*t)'` plots two
  parametric curves on one set of axes and exits 0
- [ ] `calc plot 'sin(x)' 'cos(x)'` produces a legend with entries `sin(x)` and
  `cos(x)` in matching colors (verified by presence of text in SVG output)
- [ ] `calc plot --parametric 'sin(t)'` (odd number of expressions) prints
  `error: parametric mode requires an even number of expressions` to stderr and exits 1
- [ ] `calc plot` with no expressions prints usage to stderr and exits 1
- [ ] All v0.5.x success criteria still pass with a single expression
- [ ] `make test` passes clean on macOS and Linux with no display required

---

## Scope

**Included:**

- Multiple expression arguments: each becomes a separate curve on the same axes
- Automatic color assignment: each curve gets a distinct color from a fixed palette;
  colors cycle if there are more curves than palette entries
- Legend: each curve labeled with its expression string
- Unified y-range: auto-range encompasses the full extent of all curves
- Parametric mode (`--parametric`): expressions are taken in pairs `(x(t), y(t))`;
  each pair is one parametric curve
- Parametric domain: `--tmin` / `--tmax` (default: `0` to `2π`)
- All v0.5.x flags (`--xmin`, `--xmax`, `--width`, `--height`, `--output`) apply

---

## Failure Modes

**Parametric:**
- Odd number of expressions in parametric mode → `error: parametric mode requires an even number of expressions` to stderr, exit 1
- `--xmin` / `--xmax` used with `--parametric` → `error: --xmin/--xmax not valid in parametric mode; use --tmin/--tmax` to stderr, exit 1
- `--tmin` ≥ `--tmax` → `error: tmin must be less than tmax` to stderr, exit 1

**Range:**
- All curves undefined over their entire domain → `error: all expressions undefined over domain` to stderr, exit 1

**Input validation:**
- No expressions → print usage to stderr, exit 1

---

## Interface Contracts

**CLI:**

```
calc plot '<expr1>' ['<expr2>' ...] [--xmin N] [--xmax N] [--width N] [--height N] [--output PATH]
calc plot --parametric '<x1(t)>' '<y1(t)>' ['<x2(t)>' '<y2(t)>' ...] [--tmin N] [--tmax N] [--width N] [--height N] [--output PATH]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--parametric` | off | Treat expressions as pairs `(x(t), y(t))` |
| `--tmin` | `0` | Parametric domain start |
| `--tmax` | `2π` | Parametric domain end |
| All v0.5.x flags | unchanged | Same as v0.5.x |

| Stream | Content |
|--------|---------|
| stdout | Empty on success |
| stderr | `error: <description>` on any failure |
| exit 0 | Image written successfully |
| exit 1 | Any error |

---

## Constraints

- All v0.5.x constraints remain in force
- Color palette must have at least 8 distinct, visually separable colors
- The curve data model must extend to implicit curves `F(x, y) = 0` in v0.9.x
  without requiring a separate rendering code path for axes and legends

---

## Key Unknowns

1. **[P1]** Does the curve data model — a sampled sequence of (x, y) points with
   a label and color — extend to implicitly-defined curves (where points are found
   by a root-finding or marching algorithm rather than direct sampling) without
   replacing the renderer in v0.9.x? The answer determines whether the curve
   abstraction established in v0.6.x is the permanent interface between the
   math layer and the rendering layer.

## Non-Goals

- Interactive window — v0.7.x
- In-window expression editing — v0.8.x
- Implicit curves and inequalities — v0.9.x
- Polar coordinates
- 3D plotting
- Windows support
