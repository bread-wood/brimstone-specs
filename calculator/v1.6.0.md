# Calculator CLI v1.6.x — Sliders

**Theme:** Variables you can drag. The plot becomes explorable.

## Overview

Adds parameter sliders to the interactive graphing window. A user types
`a * sin(x + b)` and the window automatically detects that `a` and `b` are
free parameters — not the plot variable `x` — and renders a slider for each.
Dragging a slider updates the plot in real time. An animate button sweeps the
parameter automatically, turning any expression into a family-of-curves
animation. This is the feature that makes the graphing calculator feel like
Desmos — the difference between showing a curve and exploring a space of curves.

---

## Success Criteria

- [ ] Entering `a * sin(x)` in the expression editor with no prior definition
  of `a` creates a slider for `a` with a default range of `[-10, 10]`
- [ ] Dragging the `a` slider updates the curve in real time with no visible lag
- [ ] Entering `a * sin(x + b)` creates sliders for both `a` and `b`
- [ ] Right-clicking a slider allows setting its min, max, and step values
- [ ] Clicking the play button on a slider animates `a` sweeping from min to max
  and back continuously; clicking again pauses
- [ ] `calc plot 'a * sin(x)' --interactive --slider a=1,min=-5,max=5` opens the
  window with `a` initialised to `1` and a slider range of `[-5, 5]`
- [ ] Defining `a = 2` explicitly in the expression list removes the auto-slider
  for `a` and uses the defined value
- [ ] `calc plot 'a * sin(x)' --interactive --screenshot out.png` renders with
  `a` at its default value and exits 0
- [ ] `make test` passes clean using `--screenshot` mode

---

## Scope

**Included:**

- Automatic slider creation: any variable referenced in an expression but not
  explicitly defined in the expression list is rendered as a slider
- Default slider range: `[-10, 10]`; default initial value: `1`
- Slider UI: labelled with variable name and current value; draggable
- Right-click slider configuration: set min, max, step, and initial value
- Animation: per-slider play/pause button; configurable sweep speed
- CLI slider initialisation: `--slider name=value,min=N,max=N`
- Explicit variable definition takes precedence over auto-slider
- All v0.8.x expression editor interactions unchanged

---

## Failure Modes

**Slider detection:**
- Expression references more than `CALC_MAX_SLIDERS` (default: 8) undefined
  variables → sliders created for the first 8 by alphabetical order; remaining
  variables produce `error: undefined variable: <name>` on evaluation; a warning
  is shown in the expression panel

**Animation:**
- Animation frame rate drops below 10 fps → animation continues at reduced
  rate; no error

---

## Interface Contracts

**CLI additions:**

```
calc plot '<expr>' --interactive [--slider <name>=<value>,min=<N>,max=<N>[,step=<N>]] ...
```

**Slider UI interactions:**

| Action | Result |
|--------|--------|
| Drag slider | Update parameter value; rerender curve |
| Right-click slider | Open configuration popup (min, max, step, value) |
| Click play button | Animate parameter from min to max and back |
| Click pause button | Stop animation at current value |
| Double-click value label | Edit value numerically |

**Environment variables:**

| Variable | Default | Purpose |
|----------|---------|---------|
| `CALC_MAX_SLIDERS` | `8` | Maximum number of auto-created sliders |
| `CALC_ANIM_FPS` | `60` | Target animation frame rate |

| Stream | Content |
|--------|---------|
| stdout | Empty |
| stderr | `error: <description>` on fatal startup failure |
| exit 0 | Window closed or `--screenshot` completed |
| exit 1 | Fatal error |

---

## Constraints

- All v0.8.x constraints remain in force
- Slider state (name, value, min, max) must be included in the session file
  schema from v1.0.x so that slider positions are preserved across sessions
- The animation mechanism must extend to parametric animation (sweeping `t` in
  parametric mode) without a separate animation system
- Slider rerender must not block the UI thread; parameter updates and curve
  resampling run off the main thread

---

## Key Unknowns

1. **[P1]** Does automatic slider creation — detecting which variables in an
   expression are "free parameters" vs. "intended to be defined elsewhere" —
   produce reliable results, or does the heuristic (undefined in expression list
   = slider) create confusion when users forget to define a variable and get a
   slider instead of an error? The answer determines whether auto-detection is
   the primary mechanism or whether sliders require explicit declaration.

---

## Non-Goals

- Linked sliders (one slider controlling multiple variables)
- Slider-driven table output (show parameter values and results in a table)
- Saving animations as video
- Windows support
