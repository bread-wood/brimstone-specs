# Calculator CLI v1.7.x — 3D

**Theme:** The plot gains a dimension. Functions of two variables become surfaces.

## Overview

Adds a `plot3d` subcommand that renders a surface `z = f(x, y)` over a
configurable 2D domain. The surface is displayed in an interactive window with
perspective rotation via mouse drag and zoom via scroll. The same renderer
handles multiple surfaces and can overlay 2D curves (cross-sections) on the
surface. This is the natural extension of the 2D graphing capability and the
version that attracts the multivariable calculus audience.

---

## Success Criteria

- [ ] `calc plot3d 'sin(x) * cos(y)'` opens a window displaying the surface
  and exits 0 when closed
- [ ] `calc plot3d 'sin(x) * cos(y)' --screenshot out.png` renders to file
  and exits 0 with no window
- [ ] `calc plot3d 'x^2 + y^2'` displays a paraboloid
- [ ] Click-and-drag rotates the surface; scroll zooms
- [ ] `calc plot3d 'sin(x) * cos(y)' --xmin -3 --xmax 3 --ymin -3 --ymax 3`
  plots over the specified domain
- [ ] `calc plot3d 'sin(x) * cos(y)' 'x + y'` renders two surfaces in distinct
  colors on the same axes
- [ ] Color maps to z-value by default; `--colormap` selects from a set of
  named palettes
- [ ] `calc plot3d 'sin(x) * cos(y)' --wireframe` renders as a mesh instead
  of a filled surface
- [ ] `calc plot3d` with no expression prints usage to stderr and exits 1
- [ ] `make test` passes clean using `--screenshot` mode

---

## Scope

**Included:**

- `calc plot3d '<expr>'` subcommand; free variables are `x` and `y`
- Configurable domain: `--xmin`, `--xmax`, `--ymin`, `--ymax` (default: `-5` to `5`)
- Interactive window: perspective projection; mouse-drag rotation; scroll zoom
- `--screenshot PATH` for headless rendering
- Multiple surfaces: multiple expression arguments
- Color mapped to z-value with configurable colormap (`--colormap viridis|plasma|grayscale`)
- Wireframe mode: `--wireframe`
- Configurable grid resolution: `--resolution N` (samples per axis, default `80`)
- All v0.6.x flags where applicable (`--output`, `--width`, `--height`)

---

## Failure Modes

**Expression:**
- Expression contains undefined variable other than `x` or `y` →
  `error: undefined variable: <name>` to stderr, exit 1
- Expression undefined over the entire domain → `error: expression undefined over entire domain` to stderr, exit 1

**Display:**
- No display available without `--screenshot` → `error: no display available` to stderr, exit 1

---

## Interface Contracts

**CLI:**

```
calc plot3d '<expr>' ['<expr>' ...] [--xmin N] [--xmax N] [--ymin N] [--ymax N]
    [--resolution N] [--wireframe] [--colormap NAME]
    [--width N] [--height N] [--output PATH] [--screenshot PATH]
```

**Mouse bindings:**

| Gesture | Action |
|---------|--------|
| Click + drag | Rotate surface |
| Scroll | Zoom |
| Right-click + drag | Pan |

| Stream | Content |
|--------|---------|
| stdout | Empty |
| stderr | `error: <description>` on failure |
| exit 0 | Window closed or `--screenshot` completed |
| exit 1 | Fatal error |

---

## Constraints

- All v0.7.x constraints remain in force for the interactive window
- The 3D renderer must use the same scene abstraction as the 2D renderer
  where possible; axes, labels, and window management are shared
- The 3D renderer does not need to support implicit surfaces `F(x,y,z) = 0`
  in this version; only explicit `z = f(x,y)`
- `--screenshot` must produce output with no display available (for CI)

---

## Key Unknowns

1. **[P1]** Does the rendering pipeline from v0.5.x — designed for 2D scenes —
   extend to a 3D perspective projection and surface rasterization without
   replacing the renderer, or does 3D require a fundamentally different rendering
   stack (e.g. OpenGL/Metal/WebGPU) that cannot share the 2D code path? The
   answer determines whether `plot3d` is an extension of the existing renderer
   or a new subsystem.

---

## Non-Goals

- Implicit 3D surfaces (`F(x,y,z) = 0`)
- Volumetric rendering
- 3D parametric curves (space curves)
- 3D vector fields
- Windows support
