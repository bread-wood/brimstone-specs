# Calculator CLI v0.9.x — Implicit

**Theme:** Equations as curves. Relations, not just functions.

## Overview

Extends the expression editor with implicit curve and inequality rendering.
A user can type `x^2 + y^2 = 25` and see a circle, or `y > sin(x)` and see a
shaded half-plane. Implicit curves require a fundamentally different rendering
approach than explicit ones — the renderer must find the zero-set of `F(x, y)`
rather than sampling `y = f(x)` — but the result integrates into the same
expression list and plot canvas as explicit curves and parametric curves. This
version validates that the curve abstraction from v0.6.x is a permanent
interface between the math layer and the rendering layer.

---

## Success Criteria

- [ ] Entering `x^2 + y^2 = 25` in the expression editor renders a circle of
  radius 5 centered at the origin
- [ ] Entering `x^2 + y^2 = 25` and `y = x` on the same plot renders the circle
  and the line together with distinct colors
- [ ] Entering `y > sin(x)` renders the region above `sin(x)` as a shaded area
  with the boundary curve visible
- [ ] Entering `y < x^2` renders the region below `y = x^2` as a shaded area
- [ ] Entering `x^2 + y^2 < 9` renders the interior of a circle as a shaded area
- [ ] `calc plot 'x^2 + y^2 = 25' --screenshot out.png` renders the circle to
  file and exits 0
- [ ] Entering `x^2 + y^2 = 0` renders a single point at the origin (degenerate case)
- [ ] Entering `x^2 + y^2 = -1` renders nothing (no real solutions); the expression
  row is highlighted with a warning: `no points in current viewport`
- [ ] Editing an implicit expression live updates the curve as in v0.8.x explicit
  editing
- [ ] All v0.8.x success criteria still pass
- [ ] `make test` passes clean on macOS and Linux using `--screenshot` mode

---

## Scope

**Included:**

- Implicit curve syntax: `F(x, y) = G(x, y)` — rendered as the zero-set of `F - G`
- Inequality syntax: `F(x, y) > G(x, y)`, `<`, `≥`, `≤` — rendered as shaded regions
  with the boundary curve
- Implicit curves and inequalities compose freely with explicit curves and parametric
  curves on the same plot
- Warning indicator (not an error) when an expression has no visible points in the
  current viewport
- Shading opacity is fixed and does not obscure underlying explicit curves

---

## Failure Modes

**Implicit evaluation:**
- Expression has no real solutions in the current viewport → expression row shows
  `no points in current viewport` warning; no curve or shading drawn; not an error
- Expression produces only isolated points (degenerate zero-set) → points rendered;
  not an error

**Input (inherited):**
- Syntax error in expression → field highlighted red; last valid rendering retained

---

## Interface Contracts

**CLI:**

No new CLI flags. Implicit expressions are passed as arguments or entered in the
editor using the same interface as explicit ones.

**Expression syntax extension:**

| Syntax | Rendering |
|--------|-----------|
| `y = f(x)` | Explicit curve (v0.5.x behavior) |
| `x(t), y(t)` (parametric mode) | Parametric curve (v0.6.x behavior) |
| `F(x,y) = G(x,y)` | Implicit curve |
| `F(x,y) > G(x,y)` | Shaded region (strict inequality, open boundary) |
| `F(x,y) >= G(x,y)` | Shaded region (non-strict inequality, closed boundary) |
| `F(x,y) < G(x,y)` | Shaded region below |
| `F(x,y) <= G(x,y)` | Shaded region below, closed |

| Stream | Content |
|--------|---------|
| stdout | Empty |
| stderr | `error: <description>` on fatal startup failure |
| exit 0 | Window closed normally or `--screenshot` completed |
| exit 1 | Fatal error |

---

## Constraints

- All v0.8.x constraints remain in force
- Implicit rendering must use the same axes, color palette, and legend as explicit
  rendering; no separate rendering code path for axes or labels
- Shading for inequalities must be visually distinct from curve color and must
  not make text labels or the cursor readout unreadable
- Implicit curve rendering resolution must be sufficient to show smooth curves at
  the default viewport; quality may degrade at extreme zoom

---

## Key Unknowns

1. **[P1]** Does the implicit curve renderer — which finds the zero-set of
   `F(x, y)` over the visible viewport — remain fast enough to support live
   editing (per-keystroke rerender) as the viewport grows or the expression
   complexity increases, without introducing a rerender delay that breaks the
   v0.8.x live-edit contract? The answer determines whether implicit curves can
   share the live-edit pipeline or require a debounced update path.

---

## Non-Goals

- 3D implicit surfaces
- Complex number visualizations (domain coloring, Riemann surfaces)
- Variable sliders
- Windows support
