# wowbot v0.3.4 — Contractor

**Theme:** Let players bring the mats. Collect the fee.

## Overview

Fulfills player-commissioned crafting orders from the Work Order board. Players
post orders specifying an item they want crafted, the mats they will supply, and
a gold fee they will pay. The bot monitors the board, evaluates each order's fee
against the time cost of fulfilling it, accepts profitable orders, collects the
supplied mats, crafts the item, and delivers it. No mat purchasing, no AH
scanning, no inventory risk. The bot is a pure service provider: other players
take the market risk; the bot collects fees for execution.

---

## Success Criteria

- [ ] `wowbot contractor --profession <prof>` scans the Work Order board, accepts
  orders with fees above `--min-fee`, crafts and delivers them, and exits 0 with
  a session report to stdout
- [ ] Session report includes: orders accepted, orders fulfilled, total fees
  collected, time per order, profession used
- [ ] The bot only accepts orders where the customer supplies all required mats
  (no self-mat orders in this version)
- [ ] `wowbot contractor --profession <prof> --dry-run` prints available orders
  with fee and estimated fulfillment time without accepting any
- [ ] An order accepted but not fulfillable (mats not received within timeout)
  is abandoned; bot logs warning and continues to next order
- [ ] `wowbot contractor` with no `--profession` flag prints
  `error: --profession is required` to stderr and exits 1
- [ ] `make test` passes clean with no live WoW instance

---

## Scope

**Included:**

- Work Order board navigation and scanning
- Order evaluation: fee, mats required (customer-supplied), estimated craft time
- Minimum fee filter: skip orders below `--min-fee`
- Order acceptance: select order, confirm acceptance
- Mat collection: retrieve mats supplied by the customer from the order interface
- Crafting execution: craft the commissioned item (single craft per order)
- Delivery: return crafted item via the order fulfillment interface
- Quality targeting: craft at the highest quality achievable with supplied mats
- Dry-run mode: evaluate and display board without accepting
- Session report: JSON written to stdout on exit

---

## Failure Modes

**Order acceptance:**
- Order claimed by another crafter between scan and accept → skip, continue to next

**Mat collection:**
- Customer-supplied mats not available within `WOWBOT_ORDER_WAIT_SECONDS` →
  abandon order; log warning; continue

**Crafting:**
- Craft fails (insufficient skill, missing tool) → abandon order; log error;
  do not deliver a failed craft

**Delivery:**
- Delivery UI not recognised → save screenshot; log error; mark order as failed
  in report; continue

## Interface Contracts

**CLI:**

```
wowbot contractor --profession <prof> [--min-fee <gold>] [--max-orders <N>] [--dry-run]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--profession` | required | Crafting profession to fulfill orders for |
| `--min-fee` | `500` | Skip orders with fees below this value |
| `--max-orders` | unlimited | Stop after fulfilling this many orders |
| `--dry-run` | off | Evaluate board without accepting |

**Session report JSON schema:**

```json
{
  "session_at": "<ISO8601>",
  "realm": "<realm-name>",
  "profession": "Alchemy",
  "orders_scanned": 24,
  "orders_accepted": 6,
  "orders_fulfilled": 5,
  "orders_abandoned": 1,
  "total_fees_collected": 8400,
  "avg_time_per_order_seconds": 47
}
```

**Environment variables:**

| Variable | Required | Default | Purpose |
|----------|----------|---------|---------|
| `WOWBOT_WOW_PATH` | No | auto | Path to WoW installation |
| `WOWBOT_LOG_DIR` | No | `~/.wowbot/logs` | Session logs |
| `WOWBOT_ORDER_WAIT_SECONDS` | No | `60` | Max wait for customer mats before abandoning |

## Constraints

- Depends on v0.3.2 (Crafter) for crafting execution primitives; Contractor
  uses the same craft-execution layer with the item specified by the order
- Customer-supplied-mats orders only in this version; self-mat orders (where
  the bot buys mats from AH) are a non-goal
- Quality targeting is best-effort; the bot cannot guarantee a specific quality
  tier if the customer supplies borderline mats
- The bot does not negotiate fees or counter-offer; it accepts or skips

---

## Key Unknowns

1. **[P1]** Does the Work Order board provide structured data (item name, required
   mats, fee) in a format extractable by screen reading — or does evaluating an
   order's profitability require navigating into each order's detail view
   individually, making batch evaluation too slow to be practical? The answer
   determines the board-scanning strategy and the practical throughput of the
   contractor loop.

2. **[P2]** Is quality control in WoW Midnight's crafting system deterministic
   given known inputs (skill level, supplied mats, inspiration procs) — allowing
   the bot to predict output quality before committing — or is it probabilistic
   in ways that create risk of delivering a lower quality item than the customer
   expects? The answer determines whether quality promises can be made at order
   acceptance time or must be disclaimed.

---

## Non-Goals

- Self-mat orders (bot purchases mats from AH)
- Negotiating fees
- Multi-craft orders (one craft per order in this version)
- Tracking order history for reputation management
- Windows support
