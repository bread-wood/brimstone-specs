# wowbot v0.5.1 — Instance

**Theme:** Enter the dungeon. Clear it with a group.

## Overview

Adds dungeon instanced content to the farming repertoire. The bot queues via
the Dungeon Finder, enters an instance with a group, follows the group's
movement through the dungeon, applies its rotation to trash and boss encounters,
rolls on loot, and exits cleanly. The bot does not lead the group — it follows,
reacts, and avoids standing in hazards. The first version capable of running
content that requires a full party.

---

## Success Criteria

- [ ] `wowbot dungeon --dungeon <name> --spec <spec>` queues via Dungeon Finder,
  enters the instance when the group forms, clears it, and exits 0 with a run
  report to stdout
- [ ] Run report includes: dungeon name, duration, bosses killed, items looted,
  gold looted, deaths, role played
- [ ] The bot follows group movement: when other players move toward the next
  pack, the bot follows rather than staying at its last position
- [ ] The bot detects and moves out of AoE ground effects (red circles, void zones)
  within one GCD of appearance
- [ ] On boss encounters the bot continues its rotation unless a dodge condition
  is detected; it does not attempt to coordinate with other players
- [ ] On a loot roll: Need on armor matching spec armor type and primary stat;
  Greed on everything else; Pass on nothing
- [ ] After 3 wipes within one dungeon run the bot exits with
  `error: repeated wipes, halting` to stderr and exits 1
- [ ] `wowbot dungeon` with no arguments prints usage to stderr and exits 1
- [ ] `make test` passes clean with no live WoW instance

---

## Scope

**Included:**

- Dungeon Finder queue: open the UI, select the configured dungeon (or random),
  queue as DPS, accept the ready check
- Group follow: detect group member positions; move toward the group centroid
  when the group advances
- AoE avoidance: detect ground effect indicators by color/shape; move out
- Boss encounter mode: continue rotation; prioritize dodge over DPS on AoE events
- Loot rolls: evaluate each item against spec armor type and primary stat;
  roll Need or Greed accordingly
- Wipe detection: all party members dead simultaneously; release, run back,
  resume on resurrect
- Repeated wipe halt: 3 wipes in one run → halt
- Run report: JSON written to stdout on exit

---

## Failure Modes

**Queue and entry:**
- Queue wait exceeds `WOWBOT_DUNGEON_QUEUE_TIMEOUT_MINUTES` → exit 1 with
  `error: queue timeout` to stderr
- Ready check missed → re-queue; log warning

**In-dungeon:**
- Bot becomes separated from the group (distance > threshold for > 30 s) →
  attempt to navigate to nearest group member; if unreachable within 60 s →
  `error: lost group` to stderr, exit 1
- Dungeon completion not detected after all bosses killed → wait 5 minutes;
  exit with partial report and a warning

**Wipes:**
- Single wipe → release, run back, resume
- 3 wipes in one run → `error: repeated wipes, halting` to stderr, exit 1

## Interface Contracts

**CLI:**

```
wowbot dungeon --spec <spec> [--dungeon <name>] [--role <role>]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--spec` | required | Class spec identifier |
| `--dungeon` | random | Specific dungeon to queue for, or `random` |
| `--role` | `dps` | Role to queue as (`dps` only in this version) |

| Stream | Content |
|--------|---------|
| stdout | Run report JSON on exit |
| stderr | `error: <description>` on fatal failure |
| exit 0 | Dungeon cleared or clean stop |
| exit 1 | Fatal error |

**Run report JSON schema:**

```json
{
  "session_at": "<ISO8601>",
  "dungeon": "The Deadmines",
  "spec": "frost-mage",
  "role": "dps",
  "duration_seconds": 1820,
  "bosses_killed": 5,
  "deaths": 1,
  "wipes": 0,
  "items_looted": ["Emberstone Staff"],
  "gold_looted": 340
}
```

**Environment variables:**

| Variable | Required | Default | Purpose |
|----------|----------|---------|---------|
| `WOWBOT_WOW_PATH` | No | auto | Path to WoW installation |
| `WOWBOT_LOG_DIR` | No | `~/.wowbot/logs` | Session logs |
| `WOWBOT_DUNGEON_QUEUE_TIMEOUT_MINUTES` | No | `30` | Max queue wait before exit |

## Constraints

- Depends on v0.4.2 for multi-spec rotation engine; dungeon spec configs use the
  same format as farming spec configs
- Depends on v0.3.0 for navigation; group follow uses the same movement primitives
- The bot queues as DPS only; tanking and healing are not supported in this version
- The bot does not communicate with other players (no chat, no marking, no leading)
- AoE avoidance operates on visual detection only; no memory reading
- macOS only in this version

---

## Key Unknowns

1. **[P1]** Can group movement be detected and followed reliably using screen-based
   position tracking of group member nameplates or unit frames — or does following
   a group through a dungeon require a fundamentally different tracking mechanism
   than the waypoint navigation used in open-world farming? The answer determines
   whether the v0.3.0 navigation primitives extend to dungeon following or require
   a new movement mode.

2. **[P2]** Can AoE ground effects be detected reliably by visual signature
   (color, shape, animation) across all dungeons without per-dungeon configuration,
   or does each dungeon's unique effect palette require a manually curated detection
   profile? The answer determines whether AoE avoidance is a general capability or
   a per-dungeon configuration burden.

---

## Non-Goals

- Tanking or healing roles
- Leading or coordinating the group
- Boss mechanic scripting beyond AoE avoidance
- Multiple dungeon runs in one session (one run per invocation in this version)
- Mythic+ keys or timed content
- Windows support
