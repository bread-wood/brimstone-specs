# wowbot v0.2.1 — AH Flipper

**Theme:** Understand the market, act on it. First version that transacts.

## Overview

Computes market prices from an AH scan, identifies underpriced listings, and executes
buy and relist cycles against them. Enforces a gold budget, collects items from the
mailbox before each session, and posts relists at market price. Ships a dry-run mode
that reports opportunities without transacting. Depends on v0.1.1 for scan output
and v0.1.0 for input primitives.

---

## Success Criteria

- [ ] `wowbot ah flip --budget 50000` reads opportunities from the AH, buys items below
  threshold, relists them at market price, and exits 0 with a session report to stdout
- [ ] Session report includes: items bought (name, price paid, quantity), items relisted
  (name, relist price, expected profit), gold spent, and gold remaining
- [ ] `wowbot ah flip --budget 50000` never spends more than 50000 gold in a single
  session regardless of race conditions on the AH
- [ ] `wowbot ah flip` with no `--budget` flag prints `error: --budget is required` to
  stderr and exits 1
- [ ] `wowbot ah flip --budget 50000` collects any items waiting in the mailbox before
  scanning, so prior-session purchases are available for relist
- [ ] `wowbot ah flip --budget 50000` when inventory is full before relist completes
  prints `error: inventory full` to stderr and exits 1
- [ ] `make test` passes clean with no live WoW instance

---

## Scope

**Included:**

- Market price computation from AH snapshot; configurable minimum listing count before
  a price estimate is trusted
- Opportunity identification: listings below `--threshold` percent of market price;
  ranked by expected profit descending
- Dry-run mode: compute and report opportunities without opening any buy or post dialogs
- Mailbox collection: open mailbox, collect all items and gold, before each session
- Buy execution: search item, select listing, confirm purchase
- Relist execution: post bought items at market price with standard AH deposit
- Budget enforcement: track cumulative gold spent; halt buying when budget reached
- Session report: JSON written to stdout on exit

---

## Failure Modes

**Buy execution:**
- Listing sells before bot can buy → skip, log miss, continue to next opportunity
- Insufficient gold → skip item, log, continue
- Buy confirmation dialog not recognised → abort buy, save screenshot, log, continue

**Relist execution:**
- Computed relist price below AH minimum → skip relist, log warning
- Inventory full → `error: inventory full` to stderr, exit 1

**Budget:**
- Budget exhausted mid-session → stop buying; complete any in-progress relist;
  write partial report; exit 0

**State:**
- Disconnect during buy → exit 1; report written with items purchased so far
- Disconnect during relist → exit 1; report notes items bought but not listed

## Interface Contracts

**CLI:**

```
wowbot ah flip --budget <gold> [--threshold <pct>] [--output PATH]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--budget` | required | Maximum gold to spend this session |
| `--threshold` | `80` | Buy listings at or below this percent of market price |
| `--output` | stdout | Path to write session report JSON |

| Stream | Content |
|--------|---------|
| stdout | Session report JSON on completion |
| stderr | `error: <description>` on any fatal failure |
| exit 0 | Session completed or budget exhausted |
| exit 1 | Fatal error: WoW not running, inventory full, disconnect |

**Session report JSON schema:**

```json
{
  "session_at": "<ISO8601>",
  "realm": "<realm-name>",
  "gold_spent": 42300,
  "budget": 50000,
  "opportunities_found": 12,
  "bought": [
    { "item_id": 111, "item_name": "Frostbloom", "quantity": 20, "paid": 100, "market": 160 }
  ],
  "relisted": [
    { "item_id": 111, "item_name": "Frostbloom", "quantity": 20, "at": 160, "expected_profit": 1200 }
  ],
  "skipped": [
    { "item_id": 222, "item_name": "Moonpetal", "reason": "sold_before_buy" }
  ]
}
```

**Environment variables:**

| Variable | Required | Purpose |
|----------|----------|---------|
| `WOWBOT_WOW_PATH` | No | Path to WoW installation; auto-detected if absent |
| `WOWBOT_LOG_DIR` | No | Session logs and error screenshots (default: `~/.wowbot/logs`) |

---

## Constraints

- Depends on v0.1.1 for AH scan output; the AH listing JSON schema from v0.1.1 must
  remain stable through this version
- Budget cap is a hard limit; gold balance must be checked before each buy attempt
- All buy and post actions use the humanized input primitives from v0.1.0
- No external price data sources

## Non-Goals

- Price history or time-series tracking
- Commodity market
- Sniper mode
- Cross-character gold transfer
