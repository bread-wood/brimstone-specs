# wowbot v0.1.1 — AH Scanner

**Theme:** Read the auction house. No buying, no selling — pure market observation.

## Overview

Navigates to the auction house, pages through all current listings, and writes structured
price data to JSON. The first version with targeted UI navigation and data extraction
beyond basic game state. Depends on v0.1.0: the screen reading and input primitives
built there are the substrate this version drives.

---

## Success Criteria

- [ ] `wowbot ah scan` opens the auction house UI, pages through all listings, writes
  `prices.json` to the current directory, and exits 0
- [ ] `prices.json` contains item ID, item name, quantity, buyout price, and time
  remaining for every listing visible during the scan
- [ ] `wowbot ah scan --output <path>` writes to the specified path and exits 0
- [ ] `wowbot ah scan --fixture <path>` produces output matching the fixture's expected
  JSON without a live WoW instance and exits 0
- [ ] `wowbot ah scan` when no auctioneer is in range prints `error: no auctioneer in
  range` to stderr and exits 1
- [ ] `wowbot ah scan` when the AH UI layout is not recognised saves a screenshot and
  prints `error: AH UI not recognised; screenshot saved to <path>` to stderr and exits 1
- [ ] `make test` passes clean with no live WoW instance

---

## Scope

**Included:**

- AH UI navigation: locate and interact with an auctioneer, open the AH panel, page
  through all listings
- Listing extraction: item name, item ID, quantity, buyout price, time remaining for
  each visible listing
- JSON output of all listings with scan metadata
- Error screenshots on extraction failure

---

## Failure Modes

**Preconditions:**
- No auctioneer in range → `error: no auctioneer in range` to stderr, exit 1

**Screen reading:**
- AH UI layout not recognised (patch change, non-default UI scale) →
  `error: AH UI not recognised; screenshot saved to <path>` to stderr, exit 1
- Listing extraction confidence below threshold → entry omitted, warning logged,
  scan continues

**Network / state:**
- AH page fails to load within timeout → retry up to 3 times; on exhaustion exit 1
  with partial output written
- Game disconnect mid-scan → exit 1; partial output written;
  `error: disconnected during scan` to stderr

## Interface Contracts

**CLI:**

```
wowbot ah scan [--output PATH] [--fixture PATH]
```

| Stream | Content |
|--------|---------|
| stdout | Absolute path to written JSON file on success |
| stderr | `error: <description>` on any failure |
| exit 0 | Scan completed; JSON written |
| exit 1 | Any error |

**Output JSON schema:**

```json
{
  "scanned_at": "<ISO8601>",
  "realm": "<realm-name>",
  "listings": [
    {
      "item_id": 12345,
      "item_name": "Frostbloom",
      "quantity": 20,
      "buyout_gold": 150,
      "time_remaining": "long"
    }
  ]
}
```

`time_remaining` values: `"short"` (< 30 min), `"medium"` (30 min–2 h),
`"long"` (2–12 h), `"very_long"` (12–48 h).

**Environment variables:**

| Variable | Required | Purpose |
|----------|----------|---------|
| `WOWBOT_WOW_PATH` | No | Path to WoW installation; auto-detected if absent |
| `WOWBOT_LOG_DIR` | No | Session logs and error screenshots (default: `~/.wowbot/logs`) |

## Constraints

- Depends on v0.1.0; screen reading and input must pass all v0.1.0 success criteria
- No memory reading; all data extracted from the game window
- Input simulation uses the humanized primitives from v0.1.0; no new input patterns
- The extraction approach must be designed to generalise to other UI contexts — combat
  unit frames, nameplates, buff frames — so the same module can serve v0.4.x without
  a parallel implementation

## Key Unknowns

1. **[P1]** Can AH listing data be reliably extracted from the Midnight UI at standard
   scales and resolutions, and will the same extraction approach work for the combat UI
   elements needed in v0.4.x — or does each UI context require an independent extraction
   strategy? The answer determines whether extraction is a shared generalised module or
   a per-context implementation.

## Non-Goals

- Market price computation or opportunity identification (v0.2.0)
- Buying or selling (v0.2.1)
- Price history or trend tracking
- Commodity market (separate Midnight UI, deferred)
- Waypoint navigation or gathering (v0.3.x)
- Combat (v0.4.x)
