# wowbot v0.3.0 — Navigator

**Theme:** Move through the world. First version with sustained autonomous movement.

## Overview

Follows a pre-recorded waypoint route by flying between points, detects when the
character is stuck, and recovers automatically. Ships a route recorder so the operator
can define routes by flying them manually. No harvesting yet — this version proves that
sustained autonomous movement works without detection or getting permanently stuck before
node interaction is added in v0.3.1. Depends on v0.1.0 for screen reading and input.

---

## Success Criteria

- [ ] `wowbot navigate --route <name>` flies the waypoint route continuously until
  stopped with Ctrl-C and exits 0
- [ ] `wowbot navigate --route <name> --duration 30m` runs for 30 minutes and exits 0
- [ ] `wowbot navigate --route <name>` skips to the next waypoint when the character's
  position is unchanged for more than 10 seconds and logs a stuck event with coordinates
- [ ] `wowbot route record --output <path>` records the player's position at regular
  intervals until stopped with Ctrl-C; exits 0 with the route written to the specified path
- [ ] `wowbot navigate --route <missing>` prints `error: route not found: <name>` to
  stderr and exits 1
- [ ] `wowbot navigate --route <name>` when the character is not mounted and cannot
  mount (e.g., in combat) logs a warning, waits for combat to end, then mounts and resumes
- [ ] `make test` passes clean with no live WoW instance

---

## Scope

**Included:**

- Waypoint navigation: fly between waypoints in recorded order; skip to nearest
  unvisited waypoint when a deviation is detected
- Stuck detection: if character position is unchanged for more than 10 seconds,
  skip to next waypoint; log stuck event with position
- Mount/dismount management: mount before navigating; detect forced dismount
  (combat, terrain); remount when conditions allow
- Route recorder: sample character position at regular intervals; save as a route file
- Route file format: human-editable, specifies zone and ordered waypoints

---

## Failure Modes

**Navigation:**
- Stuck > 10s → skip to next waypoint; log stuck event with coordinates
- Forced dismount (combat) → wait for combat to end; remount; resume route
- Phase mismatch prevents reaching waypoint → log warning; skip to next waypoint
- Repeated stuck events at same position (3 in a row) → skip two waypoints ahead;
  log warning

**Route:**
- Route file not found → `error: route not found: <name>` to stderr, exit 1
- Route file invalid → `error: invalid route file: <path>` to stderr, exit 1
- Route has fewer than 2 waypoints → `error: route must have at least 2 waypoints`
  to stderr, exit 1

## Interface Contracts

**CLI:**

```
wowbot navigate --route <name-or-path> [--duration <duration>]
wowbot route record --output <path>
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--route` | required | Route name (from `~/.wowbot/routes/`) or path to route file |
| `--duration` | unlimited | Stop after this duration (e.g. `1h`, `30m`) |

| Stream | Content |
|--------|---------|
| stdout | Navigation summary JSON on exit |
| stderr | `error: <description>` on fatal failure |
| exit 0 | Clean stop (Ctrl-C or duration elapsed) |
| exit 1 | Fatal error |

**Navigation summary JSON schema:**

```json
{
  "session_at": "<ISO8601>",
  "duration_seconds": 1803,
  "route": "vale-herbs",
  "waypoints_visited": 42,
  "stuck_events": 2
}
```

**Route file schema:**

```json
{
  "name": "vale-herbs",
  "zone": "Vale of Eternal Blossoms",
  "waypoints": [
    { "x": 51.2, "y": 34.8 },
    { "x": 53.7, "y": 31.2 }
  ]
}
```

**Environment variables:**

| Variable | Required | Purpose |
|----------|----------|---------|
| `WOWBOT_WOW_PATH` | No | Path to WoW installation; auto-detected if absent |
| `WOWBOT_LOG_DIR` | No | Session logs (default: `~/.wowbot/logs`) |
| `WOWBOT_ROUTES_DIR` | No | Route file directory (default: `~/.wowbot/routes/`) |

## Constraints

- Depends on v0.1.0; screen reading and input must pass all v0.1.0 success criteria
- Navigation uses only in-game mounted flight; no exploit movement
- Stuck detection must not false-positive during intentional slow movement
  (swimming, steep terrain, loading screens)
- Route files are human-editable
- The movement approach must extend to the combat repositioning needed in v0.4.x
  without redesigning the movement module

---

## Key Unknowns

1. **[P1]** What movement strategy avoids Warden's pattern detection across both the
   sustained route-following in v0.3.x and the shorter reactive repositioning during
   combat in v0.4.x? The answer determines the movement architecture; if the two use
   cases require fundamentally different strategies the movement module needs to be
   designed with both in mind from the start.

---

## Non-Goals

- Node detection or harvesting (v0.3.1)
- Bank runs (v0.3.1)
- Combat (v0.4.x)
- Dynamic route generation; routes are pre-recorded
- Multi-character operation
- Windows support
