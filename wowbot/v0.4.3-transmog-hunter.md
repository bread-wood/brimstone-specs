# wowbot v0.4.3 — Antiquarian

**Theme:** Farm the past. Sell nostalgia.

## Overview

Farms old instances — raids and dungeons from previous expansions, soloable
at current level — for rare cosmetic appearances that sell for thousands of gold
with little competition. The value is in long-tail scarcity: specific tints,
weapon models, and armor sets that were only available from one source in one
expansion and have never been farmable since. The bot runs a configured rotation
of instances, clears them efficiently, identifies drops against a configurable
value catalog, posts valuable items at price-floor-protected prices, and vendors
everything else. A patience play: individual runs yield little, but over weeks
the hits compound.

---

## Success Criteria

- [ ] `wowbot antiquarian --rotation <rotation> --spec <spec>` clears each
  instance in the rotation in sequence, posts drops matching the value catalog,
  vendors junk, and exits 0 with a session report
- [ ] Session report includes: instances cleared, drops found by name, items
  posted with asking price, items vendored, raw gold from vendors and bosses
- [ ] The bot detects lockout status before entering each instance; skips
  instances on lockout and logs them as pending reset
- [ ] Items in the value catalog are posted at no less than `--price-floor`
  and no more than the current AH median for that item (if listed)
- [ ] Items not in the value catalog are vendored immediately; nothing is left
  in bags at session end
- [ ] If the value catalog has no entry for a drop and AH median for that item
  exceeds `--unknown-threshold`, the item is posted; otherwise vendored
- [ ] `wowbot antiquarian --rotation <rotation> --dry-run` prints the cleared
  loot with posting/vendor decisions without submitting to the AH
- [ ] `make test` passes clean with no live WoW instance

---

## Scope

**Included:**

- Instance rotation config: ordered list of instances with entrance coordinates
  and reset cadence (daily, weekly)
- Lockout detection: check instance lockout status before entry; skip and note
  expected reset time
- Instance clear: navigate and fight through the instance using v0.4.0 combat
  engine; no specific boss mechanic handling beyond basic combat
- Loot evaluation: compare each drop against the value catalog; classify as
  post, unknown-threshold-check, or vendor
- AH price check for unknown items: single scan for current median price
- Posting: post catalog items at price-floor-protected price
- Vendor sweep: vendor all non-catalog, below-threshold items at end of run
- Session report: JSON written to stdout on exit

---

## Failure Modes

**Instance entry:**
- Instance entrance not reachable (phasing, zone requirements) →
  skip instance; log `skipped: entrance not reachable`; continue rotation
- Lockout detected → skip; note expected reset; continue

**Combat:**
- All v0.4.0 failure modes (death, stuck) apply; death in an old instance
  triggers a spirit run back to entrance
- Repeated deaths (3 within one instance) → abandon instance; exit instance;
  continue rotation with next instance

**Loot:**
- AH price check times out or fails → fall back to `--price-floor` for
  unknown items; do not halt

**Posting:**
- AH posting fails → leave item in bags; note in report; continue

## Interface Contracts

**CLI:**

```
wowbot antiquarian --rotation <name> --spec <spec> [--price-floor <gold>] [--unknown-threshold <gold>] [--dry-run]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--rotation` | required | Named instance rotation config |
| `--spec` | required | Class spec for clearing |
| `--price-floor` | `1000` | Minimum posting price for catalog items |
| `--unknown-threshold` | `5000` | Post unknown drops above this AH median; vendor below |
| `--dry-run` | off | Classify loot without posting or vendoring |

**Session report JSON schema:**

```json
{
  "session_at": "<ISO8601>",
  "spec": "frost-mage",
  "instances_cleared": ["Ulduar 25", "ICC 10"],
  "instances_skipped": [{ "name": "DS 25", "reason": "lockout", "resets_at": "<ISO8601>" }],
  "drops_posted": [
    { "name": "Corrupted Ashbringer", "asking_price": 85000, "catalog": true }
  ],
  "drops_vendored": 47,
  "vendor_gold": 1240,
  "boss_gold": 890
}
```

**Value catalog format:**

```json
[
  { "item_id": 17182, "name": "Sulfuras, Hand of Ragnaros", "floor": 50000 },
  { "item_id": 22632, "name": "Corrupted Ashbringer", "floor": 80000 }
]
```

**Environment variables:** Same as v0.4.1.

## Constraints

- Depends on v0.4.0 for combat; old instance clearing uses the standard rotation
  engine — no per-boss scripting required in this version
- Depends on v0.3.0 for navigation within instances
- Depends on v0.1.1 for AH scanning (unknown item price check) and v0.2.1 for posting
- The value catalog is a local JSON file; the bot does not fetch prices from
  external sources or automatically update the catalog
- The bot clears instances in the rotation sequentially; no parallelism across
  instances in a single session
- macOS only in this version

---

## Key Unknowns

1. **[P1]** Can old instances — many designed before current game systems existed —
   be reliably navigated and cleared using the v0.3.0 movement primitives and
   v0.4.0 combat engine, or do legacy encounter mechanics (scripted events, unusual
   terrain, forced-phase transitions) require per-instance workarounds that make
   a general "clear old content" mode impractical without a large manual
   configuration burden? The answer determines whether Antiquarian is a general
   capability or a list of specifically tested and configured instances.

2. **[P2]** Is the value of rare transmog appearances stable enough over weeks
   and months to warrant posting at high price floors and waiting for buyers —
   or does the market for cosmetic items move quickly enough that unsold
   inventory becomes a capital lock-up problem requiring active repricing? The
   answer determines the session design (post-and-forget vs. active listing
   management) and whether Antiquarian needs integration with a price history
   system.

---

## Non-Goals

- Mythic or current-expansion content (too hard to solo)
- Per-boss scripted mechanics
- Automatic value catalog updates from external price sources
- Repricing stale listings (post and forget in this version)
- Pet and mount drops (handled the same as gear drops — catalog-based)
- Windows support
