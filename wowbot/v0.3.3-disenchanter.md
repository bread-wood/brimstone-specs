# wowbot v0.3.3 — Disenchanter

**Theme:** Buy gear. Destroy it. Sell what's inside.

## Overview

Exploits the persistent spread between gear prices and enchanting mat prices.
Gear and mats are valued by different player populations — buyers who need
gear upgrades and buyers who need crafting reagents — and these valuations
move independently, creating arbitrage opportunities. The bot scans gear
listings, computes expected disenchant yield value from item level and quality,
buys when the spread is positive, destroys the gear, and posts the resulting
mats. No mob killing. No gathering. Pure market inefficiency extraction.

---

## Success Criteria

- [ ] `wowbot de-arb --budget 50000` scans gear listings on the AH, identifies
  items where expected DE value exceeds cost by at least `--margin`, buys them,
  disenchants them, posts the mats, and exits 0 with a session report to stdout
- [ ] Session report includes: items bought (name, ilvl, cost), mats obtained
  (name, quantity), gold spent, expected profit from posted mats
- [ ] The bot never buys an item where expected DE value minus AH cut minus cost
  is below `--margin`
- [ ] `wowbot de-arb --budget 50000 --dry-run` prints the opportunity list with
  expected profit without executing any purchases
- [ ] `wowbot de-arb --budget 50000` never spends more than the budget in one
  session regardless of race conditions
- [ ] `wowbot de-arb` with no `--budget` flag prints `error: --budget is required`
  to stderr and exits 1
- [ ] `wowbot de-arb --budget 50000` when inventory is full prints
  `error: inventory full` to stderr and exits 1
- [ ] `make test` passes clean with no live WoW instance

## Scope

**Included:**

- AH scan for gear: filter by armor type, item level range, and quality
  (configurable; defaults to all equippable gear)
- DE yield model: compute expected mat yield from item level and quality using
  the known DE yield formula; no empirical sampling required
- Mat price lookup: query current AH prices for all relevant enchanting mats
- Opportunity ranking: sort by expected profit descending; enforce `--margin` floor
- Budget enforcement: track cumulative spend; halt buying when budget reached
- Disenchant execution: select item, open disenchanting UI, confirm
- Mat posting: post obtained mats at current market price
- Dry-run mode: compute and display opportunity list without transacting
- Session report: JSON written to stdout on exit

---

## Failure Modes

**Buy execution:**
- Item sells before bot buys → skip, log miss, continue
- Expected mats do not match actual yield (formula deviation) → post what was
  obtained; note discrepancy in report
- Inventory full before all items purchased → stop buying; disenchant and post
  what's in bags; report notes early stop

**Disenchant:**
- Disenchant UI not recognised → skip item, save screenshot, log warning, continue
- Item is soulbound (cannot DE) → skip, log warning

**Budget:**
- Budget exhausted mid-session → stop buying; complete any in-progress DE and
  posting; write partial report; exit 0

**State:**
- Disconnect during buy → exit 1; report items purchased but not yet DE'd

## Interface Contracts

**CLI:**

```
wowbot de-arb --budget <gold> [--margin <gold>] [--ilvl-min <N>] [--ilvl-max <N>] [--dry-run]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--budget` | required | Max gold to spend this session |
| `--margin` | `100` | Minimum profit per item after AH cut |
| `--ilvl-min` | `1` | Only scan gear at or above this item level |
| `--ilvl-max` | `999` | Only scan gear at or below this item level |
| `--dry-run` | off | Print opportunities without transacting |

**Session report JSON schema:**

```json
{
  "session_at": "<ISO8601>",
  "realm": "<realm-name>",
  "gold_spent": 18400,
  "budget": 50000,
  "items_disenchanted": [
    { "name": "Runed Cloak", "ilvl": 580, "quality": "rare", "cost": 800,
      "mats_obtained": { "Resonite Dust": 3 }, "expected_profit": 240 }
  ],
  "mats_posted": { "Resonite Dust": 9 },
  "expected_total_profit": 720
}
```

**Environment variables:** Same as v0.2.1.

## Constraints

- Depends on v0.1.1 for AH scanning and v0.2.1 for AH posting primitives
- Depends on v0.1.0 for input primitives (disenchant UI interaction)
- Requires enchanting on the character; the bot does not check whether the
  character has enchanting — this is operator responsibility
- DE yield model must be configurable (mat tables change between patches);
  yield tables are data files, not hardcoded
- The bot must not DE equipped items; only items in bags

---

## Key Unknowns

1. **[P1]** Is the disenchant yield formula deterministic given item level and
   quality — allowing the bot to compute exact expected value before buying — or
   is yield probabilistic in WoW Midnight, requiring empirical sampling of actual
   yields to build a reliable expected value model? The answer determines whether
   the opportunity evaluator can be purely predictive or needs a calibration phase.

2. **[P2]** Do the enchanting mat prices observed on a single AH scan provide
   sufficient liquidity signal to estimate whether all obtained mats will actually
   sell at the observed price — or does the volume of mats the bot posts in a
   single session move the mat market enough to invalidate the profit estimate?
   The answer determines whether the bot needs a sell-side volume model or can
   treat mat prices as fixed.

---

## Non-Goals

- Crafting with the obtained mats (selling raw mats only)
- Tracking mat price history — v0.7.0
- Milling or prospecting (parallel professions; same pattern, different version)
- Windows support
