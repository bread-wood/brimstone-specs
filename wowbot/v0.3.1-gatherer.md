# wowbot v0.3.1 — Gatherer

**Theme:** Detect nodes, harvest them, bank when full.

## Overview

Adds resource node detection and harvest interaction on top of the navigation loop from
v0.3.0. When a node is detected near the route, the bot deviates to harvest it and then
resumes. Banks when inventory reaches a threshold. Depends on v0.3.0 for movement and
v0.1.0 for screen reading and input.

---

## Success Criteria

- [ ] `wowbot gather --route vale-herbs` flies the route, detects and harvests herb nodes,
  and runs until stopped with Ctrl-C; exits 0 with yield report to stdout
- [ ] `wowbot gather --route vale-herbs --duration 1h` runs for one hour and exits 0
  with yield report
- [ ] Yield report includes: nodes harvested by type, items collected by name, gold
  looted, bank runs made, and session duration
- [ ] `wowbot gather --route vale-herbs` returns to the configured bank NPC when
  inventory reaches 80% capacity, deposits gathered items, and resumes the route
- [ ] `wowbot gather --route <path> --fixture <path>` harvests ≥ 90% of the nodes
  present in the fixture and exits 0
- [ ] `wowbot gather` when the gathering profession is not trained on the current
  character prints `error: gathering profession not found` to stderr and exits 1
- [ ] `wowbot gather` when the bank NPC is not reachable prints `error: bank NPC not
  found` to stderr and exits 1
- [ ] `make test` passes clean with no live WoW instance

---

## Scope

**Included:**

- Node detection: detect resource nodes via minimap tracking and world-space visuals
- Route deviation: when a node is detected off-route, navigate to it; return to route
  after harvest
- Harvest interaction: dismount, face node, cast gather skill, remount
- Inventory monitoring: track bag occupancy; trigger bank run at configurable threshold
- Bank run: navigate to configured bank NPC, deposit gathered items, return to route
- Yield report: JSON written to stdout on exit

---

## Failure Modes

**Node interaction:**
- Node despawns before harvest completes → resume route; log miss
- Harvest cast interrupted (out of range, facing) → retry once; on second failure skip
- Node occupied by another player → skip; continue

**Inventory / bank:**
- Bank NPC not found → `error: bank NPC not found` to stderr, exit 1
- Bank full → `error: bank full` to stderr, exit 1

**Profession:**
- Gathering profession not trained → `error: gathering profession not found` to stderr,
  exit 1 before route begins

## Interface Contracts

**CLI:**

```
wowbot gather --route <name-or-path> [--duration <duration>] [--bank-threshold <pct>] [--fixture PATH]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--route` | required | Route name or path |
| `--duration` | unlimited | Stop after this duration |
| `--bank-threshold` | `80` | Trigger bank run when bags reach this % full |
| `--fixture` | off | Replay a recorded session for testing |

| Stream | Content |
|--------|---------|
| stdout | Yield report JSON on exit |
| stderr | `error: <description>` on fatal failure |
| exit 0 | Clean stop |
| exit 1 | Fatal error |

**Yield report JSON schema:**

```json
{
  "session_at": "<ISO8601>",
  "duration_seconds": 3612,
  "realm": "<realm-name>",
  "route": "vale-herbs",
  "nodes_harvested": { "Frostbloom": 42, "Twilight Petal": 18 },
  "items_collected": { "Frostbloom": 168, "Twilight Petal": 72 },
  "gold_looted": 230,
  "bank_runs": 2,
  "stuck_events": 1
}
```

**Environment variables:**

| Variable | Required | Purpose |
|----------|----------|---------|
| `WOWBOT_WOW_PATH` | No | Path to WoW installation; auto-detected if absent |
| `WOWBOT_LOG_DIR` | No | Session logs and error screenshots (default: `~/.wowbot/logs`) |
| `WOWBOT_ROUTES_DIR` | No | Route file directory (default: `~/.wowbot/routes/`) |

## Constraints

- Depends on v0.3.0 for navigation; all stuck detection and mount management is
  inherited from there
- Depends on v0.1.0 for screen reading and input
- Harvest timing must be humanized; no fixed cast-to-remount delay
- No combat; if forced into combat skip the node and resume route
- The node detection approach must be designed with the combat UI detection needs of
  v0.4.x in mind; if the two detection contexts are incompatible the detection module
  needs separate implementations

---

## Key Unknowns

1. **[P1]** Can resource nodes be reliably detected from minimap and world-space visuals
   in Midnight, and does the detection approach generalise to the nameplate and unit
   frame detection needed for combat targeting in v0.4.x — or do the two contexts
   require separate detection implementations? The answer determines whether game element
   detection is a shared module or must be built twice.

---

## Non-Goals

- Combat during gathering; skip contested nodes
- Skinning, fishing, or archeology
- Dungeon or instanced gathering
- Dynamic route generation
- Windows support
