# wowbot v0.5.0 — Climber

**Theme:** Gain levels. The bot grows a character.

## Overview

Extends the farming loop with level-aware progression: detects the character's
current level, kills mobs appropriate for that level, detects level-ups and
re-evaluates targets, and advances to the next configured zone when current mobs
no longer yield meaningful XP. Handles kill quests opportunistically — if a kill
quest is active and its objective matches current targets, the bot navigates to
the quest giver and turns it in. The first version that can take a character from
a starting zone to a configured target level without human intervention.

---

## Success Criteria

- [ ] `wowbot level --zone-chain <chain> --spec <spec> --target-level <N>` kills
  mobs, accumulates XP, advances zones, reaches level N, and exits 0 with a
  level report to stdout
- [ ] Level report includes: starting level, ending level, zones visited, mobs
  killed per zone, deaths, duration, XP/hour average
- [ ] On level-up, the bot detects the new level, updates its mob target
  evaluation, and continues farming without human intervention
- [ ] When mobs in the current zone yield diminished XP (below configured
  threshold), the bot moves to the next zone in the chain
- [ ] If a kill quest is active with an objective matching current farm targets,
  the bot tracks kill count and navigates to the quest NPC to turn it in when
  complete, then returns to farming
- [ ] `wowbot level --zone-chain <chain> --spec <spec>` with no `--target-level`
  farms until manually stopped (Ctrl-C); exits 0 with a partial level report
- [ ] After 3 deaths in 5 minutes the bot halts with `error: repeated deaths,
  halting` to stderr and exits 1 (inherited from v0.4.1)
- [ ] `wowbot level` with no arguments prints usage to stderr and exits 1
- [ ] `make test` passes clean with no live WoW instance

## Scope

**Included:**

- Zone chain config: ordered list of zones with mob targets per zone
- Level detection: read current level from game state
- Mob target evaluation: prefer mobs within ±3 levels of player for optimal XP
- XP rate tracking: monitor XP gain rate; trigger zone advancement when rate
  falls below configured threshold
- Zone advancement: navigate to the next zone in the chain using v0.3.0 navigation
- Kill quest detection: scan active quests for kill objectives matching current
  mob types; track progress
- Quest turn-in: navigate to quest giver NPC, turn in, return to farm position
- Level report: JSON written to stdout on exit
- Death recovery: inherited from v0.4.1
- Combat and loot: inherited from v0.4.0

---

## Failure Modes

**Zone chain:**
- Zone chain config not found → `error: zone chain not found: <name>` to stderr, exit 1
- Final zone in chain reached but target level not yet met → continue farming in
  final zone; emit warning: `zone chain exhausted at level <N>; continuing in <last-zone>`
- Navigation to next zone fails → `error: cannot reach zone: <name>` to stderr, exit 1

**Quest turn-in:**
- Quest NPC not reachable → skip turn-in; log warning; continue farming
- Turn-in dialog not recognized → skip turn-in; save screenshot; log warning; continue

**Death recovery (inherited from v0.4.1):**
- Repeated deaths (3 in 5 min) → `error: repeated deaths, halting` to stderr, exit 1

## Interface Contracts

**CLI:**

```
wowbot level --zone-chain <name> --spec <spec> [--target-level <N>] [--xp-threshold <pct>]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--zone-chain` | required | Named zone progression sequence |
| `--spec` | required | Class spec identifier |
| `--target-level` | unlimited | Stop when this level is reached |
| `--xp-threshold` | `50` | Advance zone when XP rate falls below this % of zone peak |

| Stream | Content |
|--------|---------|
| stdout | Level report JSON on exit |
| stderr | `error: <description>` on fatal failure |
| exit 0 | Target level reached or clean stop |
| exit 1 | Fatal error |

**Level report JSON schema:**

```json
{
  "session_at": "<ISO8601>",
  "duration_seconds": 7200,
  "realm": "<realm-name>",
  "spec": "frost-mage",
  "level_start": 10,
  "level_end": 20,
  "zones": [
    { "zone": "Duskwood", "mobs_killed": 210, "level_start": 10, "level_end": 15 },
    { "zone": "Stranglethorn Vale", "mobs_killed": 310, "level_start": 15, "level_end": 20 }
  ],
  "deaths": 2,
  "quests_completed": 3,
  "xp_per_hour_avg": 28400
}
```

**Environment variables:** Same as v0.4.1.

## Constraints

- Depends on v0.4.2 for multi-spec support; zone chains must be spec-agnostic
  (same chain works for any spec)
- Depends on v0.4.1 for death recovery; all death recovery behavior is inherited
- Depends on v0.3.0 for zone-to-zone navigation
- Zone chain config must remain extensible to dungeon sequences in v0.5.1 without
  schema changes to the chain format itself
- macOS only in this version

---

## Key Unknowns

1. **[P1]** How does the bot determine when to advance to the next zone —
   by tracking XP gain rate over a rolling window, by comparing mob level to
   player level, or by a fixed time-in-zone threshold — and does the chosen
   approach produce zone transitions that are indistinguishable from a player's
   natural progression decisions? The answer determines the zone advancement
   algorithm and the zone chain config schema.

2. **[P2]** Can kill quest objectives be reliably detected and tracked from
   the game's quest log UI without requiring a separate quest-tracking module,
   or does quest integration require enough new screen-reading surface area to
   warrant its own version? The answer determines whether quest turn-ins are
   in scope for this version or deferred.

---

## Non-Goals

- Quest chains (accept new quests, follow quest story) — quest turn-in only
- Dungeon grinding — v0.5.1
- PvP leveling
- Crafting profession leveling
- Windows support
