# wowbot v0.4.1 — Farmer

**Theme:** Sustained farming. Die, recover, keep going.

## Overview

Wraps the combat engine from v0.4.0 in a sustained farming loop: roam a configured zone,
find mobs, kill them, loot them, bank when full, recover from death, and repeat. The
first version that can run unattended for hours. Depends on v0.4.0 for the combat engine
and v0.3.0 for movement.

---

## Success Criteria

- [ ] `wowbot farm --zone <zone> --spec <spec> --duration 1h` kills mobs, loots them,
  runs for one hour, and exits 0 with a yield report to stdout
- [ ] Yield report includes: mobs killed, items looted by name, gold looted, deaths,
  bank runs, and session duration
- [ ] The bot releases spirit and runs back to its corpse after death, then resumes
  farming without human intervention
- [ ] After 3 deaths within 5 minutes the bot halts with `error: repeated deaths,
  halting` to stderr and exits 1
- [ ] `wowbot farm --zone <zone> --spec <spec>` returns to the configured bank NPC when
  inventory reaches 80% capacity, deposits items, and resumes farming
- [ ] `wowbot farm` with no arguments prints usage to stderr and exits 1
- [ ] `make test` passes clean with no live WoW instance

---

## Scope

**Included:**

- Pull camp navigation: operator defines one or more pull positions within the zone;
  the bot cycles between them using the navigation primitives from v0.3.0, targeting
  available mobs at each position before moving to the next
- Combat and loot: delegate to the combat engine from v0.4.0
- Death recovery: release spirit; run from graveyard to corpse; resurrect; resume
- Repeated death detection: halt if 3 deaths occur within 5 minutes
- Inventory management: bank run when bags reach threshold; return to farming after
- Yield report: JSON written to stdout on exit

---

## Failure Modes

**Death recovery:**
- Graveyard spirit healer not found → `error: graveyard not found` to stderr, exit 1
- Corpse run blocked (terrain, phasing) → retry up to 3 times; on exhaustion
  `error: cannot reach corpse` to stderr, exit 1
- Repeated deaths (3 in 5 minutes) → `error: repeated deaths, halting` to stderr, exit 1

**Inventory / bank:**
- Bank NPC not reachable from farm zone → `error: bank not configured for <zone>`
  to stderr, exit 1
- Bank full → `error: bank full` to stderr, exit 1

## Interface Contracts

**CLI:**

```
wowbot farm --zone <zone> --spec <class-spec> [--duration <duration>] [--bank-threshold <pct>]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--zone` | required | Zone to farm |
| `--spec` | required | Class spec identifier |
| `--duration` | unlimited | Stop after this duration |
| `--bank-threshold` | `80` | Trigger bank run when bags reach this % full |

| Stream | Content |
|--------|---------|
| stdout | Yield report JSON on exit |
| stderr | `error: <description>` on fatal failure |
| exit 0 | Clean stop (Ctrl-C or duration elapsed) |
| exit 1 | Fatal error |

**Yield report JSON schema:**

```json
{
  "session_at": "<ISO8601>",
  "duration_seconds": 3605,
  "realm": "<realm-name>",
  "zone": "The Ringing Deeps",
  "spec": "frost-mage",
  "mobs_killed": 312,
  "deaths": 1,
  "gold_looted": 1840,
  "items_looted": { "Resonite Crystal": 44, "Ironclaw Fang": 91 },
  "bank_runs": 3
}
```

**Environment variables:**

| Variable | Required | Purpose |
|----------|----------|---------|
| `WOWBOT_WOW_PATH` | No | Path to WoW installation; auto-detected if absent |
| `WOWBOT_LOG_DIR` | No | Session logs and error screenshots (default: `~/.wowbot/logs`) |

## Constraints

- Depends on v0.4.0 for the combat engine; all rotation and targeting behaviour is
  inherited from there
- Depends on v0.3.0 for navigation; zone roaming uses the same movement module as
  route-based gathering
- Bot must not interact with other players' corpses or interfere with their combat
- Death recovery must not corpse-camp
- macOS only in this version

---

## Key Unknowns

1. **[P1]** What pull camp strategy — fixed position, short patrol, or cycling between
   multiple named positions — produces the best kill rate while generating movement
   patterns indistinguishable from legitimate play? The answer determines the zone
   configuration format and the movement behaviour between pulls.

2. **[P2]** Does Warden detect sustained farming sessions by session duration, kill rate
   regularity, movement patterns, or a combination — and what safe session length and
   activity cadence avoids triggering behavioral flags? The answer determines the
   session management policy (max duration, rest intervals) for the farming loop.

---

## Non-Goals

- Multiple specs (v0.4.2)
- Dungeon or group content
- PvP
- Elite mob or boss farming
- Dynamic mob priority (hard-coded zone mob list only in this version)
- Windows support
