# breadwinner v0.5.0 — Ticker Tape

**Theme:** Video-to-watchlist ingestion pipeline — extract investment signals from YouTube
financial content, validate them against live market data, and deliver a clean ticker
watchlist ready for breadwinner's planning pipeline.

---

## Overview

Financial YouTube channels surface investment themes, sectors, and tickers faster than
any screener. This milestone builds an ingestion pipeline that converts a YouTube video
URL into a validated, tradeable watchlist: transcript extraction → LLM thematic
analysis → market-data filtering → Alpaca watchlist delivery.

The pipeline runs as a breadwinner CLI command (`bw ingest`) and integrates with
Alpaca's existing asset and watchlist APIs (already wired in breadwinner). Output is
a named Alpaca watchlist that feeds directly into breadwinner's existing planning
pipeline (`bw plan`).

No paid transcription services are required. Financial validation uses Alpaca's asset
data (already integrated). LLM analysis uses Claude via the Anthropic SDK (already
used in breadwinner's planning stack).

---

## Success Criteria

- [ ] `bw ingest --source=youtube --url=<url> --watchlist=<name>` exits 0 and creates
  or updates a named Alpaca paper watchlist containing only validated tickers
- [ ] The command prints a structured summary: tickers extracted, tickers filtered (with
  reason), and final watchlist contents
- [ ] Tickers with market cap below $2B are filtered out and logged with reason
  `market_cap_too_small`
- [ ] Tickers not found in Alpaca's asset universe (not tradeable on Alpaca) are
  filtered out and logged with reason `not_tradeable`
- [ ] When no transcript is available for the video, the command exits non-zero with a
  clear error message and no partial watchlist is written
- [ ] When the LLM returns tickers not found in Alpaca's asset list, they are treated
  as unresolvable and logged — not silently dropped
- [ ] Ingestion results are written to `~/.breadwinner/ingest/<date>-<watchlist>.json`
  for audit purposes
- [ ] `make test && make lint` pass on mainline

## Scope

**Included:**

- `bw ingest` CLI command with `--source`, `--url`, `--watchlist` flags
- YouTube transcript extraction using `youtube-transcript-api` (auto-generated captions
  supported; manual captions preferred when available)
- LLM thematic extraction prompt: identifies investment themes, explicitly mentioned
  tickers/companies, and infers secondary sector beneficiaries not named but logically
  linked to the stated theme
- Financial validation filter using Alpaca asset data:
  - Tradeable on Alpaca (fractionable or standard equity)
  - Market cap ≥ $2B (sourced from Alpaca asset metadata or a lightweight financial
    data provider — research stage must determine which)
- Alpaca watchlist creation/update (upsert by name) using existing Alpaca client
- Audit log: per-run JSON record at `~/.breadwinner/ingest/<date>-<watchlist>.json`
  containing raw LLM output, all candidates, filter decisions, and final watchlist
- `--dry-run` flag: prints the proposed watchlist without writing to Alpaca
- `--append` flag: adds tickers to an existing watchlist rather than replacing it

**Explicitly excluded:**

- Non-YouTube video sources (v0.6.0 scope)
- Real-time or scheduled ingestion (v0.6.0 scope; this milestone is manual/on-demand only)
- Automated trading or plan generation from ingested watchlists (that is `bw plan`'s job)
- Sentiment scoring beyond Alpaca asset metadata
- TradingView integration (Alpaca watchlists are sufficient for planning pipeline input)
- Earnings calendar filtering (v0.6.0 scope)
- Speaker diarization or host-vs-guest attribution

---

## Failure Modes

**Transcript extraction:**
- Video has no transcript (private video, transcript disabled) → exit non-zero, print
  clear message, write nothing to Alpaca or audit log
- Transcript is auto-generated with low confidence → proceed; log a warning that
  transcript quality is uncertain
- `youtube-transcript-api` rate-limited or blocked → exit non-zero with actionable
  error; do not retry silently

**LLM analysis:**
- LLM returns no tickers → log warning; exit 0 with empty watchlist; write audit log
- LLM returns malformed JSON → retry once with a stricter prompt; if second attempt
  fails, exit non-zero and log raw LLM output to audit file
- LLM hallucinates a ticker not in Alpaca's universe → log with reason
  `ticker_not_found_in_alpaca`; do not silently skip

**Financial validation:**
- Alpaca asset API unavailable → abort; do not write a partially-validated watchlist
- Market cap data unavailable for a given ticker → include ticker in output but flag
  as `market_cap_unknown`; do not silently filter it

**Watchlist write:**
- Alpaca watchlist API returns error → log full response; exit non-zero; leave existing
  watchlist unchanged

## Interface Contracts

**`bw ingest` CLI:**

```
bw ingest --source=youtube --url=<youtube-url> --watchlist=<name> [--dry-run] [--append]
```

**Audit log schema (`~/.breadwinner/ingest/<date>-<watchlist>.json`):**

```json
{
  "schema_version": "1",
  "ingested_at": "<iso8601>",
  "source": "youtube",
  "url": "<str>",
  "watchlist_name": "<str>",
  "transcript_length_chars": 0,
  "llm_raw_output": "<str>",
  "candidates": [
    {
      "ticker": "<str>",
      "source": "explicit | inferred",
      "theme": "<str>",
      "disposition": "accepted | filtered",
      "filter_reason": "market_cap_too_small | not_tradeable | ticker_not_found_in_alpaca | market_cap_unknown | null"
    }
  ],
  "final_watchlist": ["<ticker>"]
}
```

**LLM extraction prompt output (structured JSON, parsed by ingestion layer):**

```json
{
  "themes": ["<str>"],
  "tickers": [
    {
      "ticker": "<str>",
      "company": "<str>",
      "source": "explicit | inferred",
      "theme": "<str>",
      "rationale": "<str>"
    }
  ]
}
```

## Constraints

- No paid transcription APIs — `youtube-transcript-api` (MIT licensed) only
- No direct Alpaca broker API calls in the ingestion layer — use breadwinner's existing
  Alpaca client abstraction
- LLM calls use the Anthropic SDK (already a breadwinner dependency); do not add
  competing LLM SDKs
- Market cap threshold ($2B) is configurable via breadwinner config, not hardcoded
- Audit logs are append-only; never overwrite a previous run's log file
- `bw ingest` must not trigger any trade or plan execution — it is read + watchlist-write only

---

## Key Unknowns

1. **[P1]** Does Alpaca's asset API return market cap data, or is a separate financial
   data source required? If Alpaca does not expose market cap on its asset records,
   breadwinner must integrate a lightweight financial data provider (e.g. yfinance, a
   free tier of a market data API). The answer determines whether this milestone
   introduces a new dependency. Research stage must verify Alpaca's asset payload
   and identify the fallback if needed.

2. **[P2]** How reliably does `youtube-transcript-api` handle auto-generated captions
   for financial content? If accuracy is too low to extract valid ticker symbols,
   the LLM prompt strategy must compensate (e.g. fuzzy company-name matching rather
   than ticker extraction). Research stage should test against 3–5 representative
   financial videos and report failure rate.

---

## Non-Goals

- Scheduled / cron-based ingestion (v0.6.0)
- Non-YouTube sources: podcasts, earnings call transcripts, SEC filings (v0.6.0)
- Automated plan generation triggered by ingestion
- Sentiment scoring or IV-rank filtering at ingestion time
- Speaker attribution or host-vs-guest signal weighting
- TradingView watchlist sync
