# breadwinner v0.5.0 — Ticker Tape

**Theme:** Video-to-watchlist ingestion pipeline — extract investment signals from YouTube
financial content, validate them against live market data, and deliver a clean ticker
watchlist ready for breadwinner's planning pipeline.

---

## Overview

Financial YouTube channels surface investment themes, sectors, and tickers faster than
any screener. This milestone builds an ingestion pipeline that converts a YouTube video
URL into a validated, tradeable watchlist: video decomposition → thematic analysis →
market-data filtering → Alpaca watchlist delivery.

The pipeline runs as a breadwinner CLI command (`bw ingest`) and integrates with
Alpaca's existing asset and watchlist APIs (already wired in breadwinner). Output is
a named Alpaca watchlist that feeds directly into breadwinner's existing planning
pipeline (`bw plan`).

**Video decomposition** is the central research question: the pipeline can extract a
text transcript (cheap, available, text-only) or submit the video directly to a
multimodal model such as Gemini (richer — captures charts, slides, visual annotations,
and speaker audio — but introduces a new API dependency and per-video cost). Research
stage must resolve this before design begins (see Key Unknowns). The decomposition
step produces a structured intermediate representation that feeds the thematic analysis
stage; the downstream stages are identical regardless of decomposition method.

**Thematic analysis** — identifying investment themes, explicit tickers, and inferred
sector beneficiaries — is performed via the Anthropic SDK (already a breadwinner
dependency). If Gemini handles decomposition, Claude handles analysis; if transcript
extraction handles decomposition, Claude handles both decomposition and analysis in a
single prompt.

Financial validation uses Alpaca's existing asset data. Validated tickers are written
to a named Alpaca watchlist.

## Success Criteria

- [ ] `bw ingest --source=youtube --url=<url> --watchlist=<name>` exits 0 and creates
  or updates a named Alpaca paper watchlist containing only validated tickers
- [ ] The command prints a structured summary: tickers extracted, tickers filtered (with
  reason), and final watchlist contents
- [ ] Tickers with market cap below $2B are filtered out and logged with reason
  `market_cap_too_small`
- [ ] Tickers not found in Alpaca's asset universe (not tradeable on Alpaca) are
  filtered out and logged with reason `not_tradeable`
- [ ] When the video cannot be decomposed (unavailable video, API failure, no transcript
  and no multimodal fallback), the command exits non-zero with a clear error message
  and no partial watchlist is written
- [ ] When the LLM returns tickers not found in Alpaca's asset list, they are treated
  as unresolvable and logged — not silently dropped
- [ ] Ingestion results are written to `~/.breadwinner/ingest/<date>-<watchlist>.json`
  for audit purposes
- [ ] `bw ingest --from-queue` reads pending entries from the breadmin-youtube queue
  store and processes each, updating queue entry status to `done` or `error`
- [ ] `make test && make lint` pass on mainline

## Scope

**Included:**

- `bw ingest` CLI command with `--source`, `--url`, `--watchlist`, `--from-queue` flags
- Video decomposition layer (method determined by research stage — see Key Unknowns):
  - **Transcript path**: extract text transcript via `youtube-transcript-api`;
    auto-generated captions supported, manual preferred
  - **Multimodal path**: submit video to Gemini API (video URL or downloaded file);
    extract structured themes, tickers, and visual context (charts, slides, on-screen
    text) in a single API call
  - Implementation ships one method as primary with the other as a declared fallback
    (or one method only, if research deems the other unnecessary)
- Thematic extraction: LLM prompt (Claude via Anthropic SDK) that identifies investment
  themes, explicitly mentioned tickers/companies, and infers secondary sector
  beneficiaries not named but logically linked to the stated theme; structured JSON output
- Financial validation filter using Alpaca asset data:
  - Tradeable on Alpaca (fractionable or standard equity)
  - Market cap ≥ $2B (sourced from Alpaca asset metadata or lightweight provider —
    research stage must determine which; see Key Unknowns)
- Alpaca watchlist creation/update (upsert by name) using existing Alpaca client
- Audit log: per-run JSON record at `~/.breadwinner/ingest/<date>-<watchlist>.json`
  containing decomposition method used, raw LLM output, all candidates, filter
  decisions, and final watchlist
- `--dry-run` flag: prints the proposed watchlist without writing to Alpaca
- `--append` flag: adds tickers to an existing watchlist rather than replacing it

**Explicitly excluded:**

- Non-YouTube video sources (v0.6.0 scope)
- Real-time or scheduled ingestion (breadmin-jobman v0.4.0-signal-grid wires scheduling)
- Automated trading or plan generation from ingested watchlists
- Sentiment scoring beyond Alpaca asset metadata
- TradingView integration
- Earnings calendar filtering (v0.6.0 scope)

## Failure Modes

**Video decomposition:**
- Transcript unavailable AND multimodal path not configured → exit non-zero; write
  nothing to Alpaca or audit log
- Gemini API unavailable → if transcript fallback is configured, use it and log warning;
  otherwise exit non-zero
- `youtube-transcript-api` rate-limited → exit non-zero; do not retry silently
- Gemini returns empty or unparseable response → retry once; if second attempt fails,
  exit non-zero and log raw response to audit file

**Thematic analysis (Claude):**
- LLM returns no tickers → log warning; exit 0 with empty watchlist; write audit log
- LLM returns malformed JSON → retry once with stricter prompt; if second fails, exit
  non-zero and log raw output to audit file
- LLM hallucinates ticker not in Alpaca universe → log with reason
  `ticker_not_found_in_alpaca`; do not silently skip

**Financial validation:**
- Alpaca asset API unavailable → abort; do not write a partially-validated watchlist
- Market cap data unavailable for a ticker → include ticker but flag as
  `market_cap_unknown`; do not silently filter it

**Watchlist write:**
- Alpaca watchlist API returns error → log full response; exit non-zero; leave existing
  watchlist unchanged

## Interface Contracts

**`bw ingest` CLI:**

```
bw ingest --source=youtube --url=<youtube-url> --watchlist=<name> [--dry-run] [--append]
bw ingest --from-queue [--queue-db=<path>] [--watchlist=<name>]
```

**Audit log schema (`~/.breadwinner/ingest/<date>-<watchlist>.json`):**

```json
{
  "schema_version": "1",
  "ingested_at": "<iso8601>",
  "source": "youtube",
  "url": "<str>",
  "watchlist_name": "<str>",
  "decomposition_method": "transcript | gemini_multimodal",
  "decomposition_input_length": 0,
  "llm_raw_output": "<str>",
  "candidates": [
    {
      "ticker": "<str>",
      "source": "explicit | inferred",
      "theme": "<str>",
      "disposition": "accepted | filtered",
      "filter_reason": "market_cap_too_small | not_tradeable | ticker_not_found_in_alpaca | market_cap_unknown | null"
    }
  ],
  "final_watchlist": ["<ticker>"]
}
```

**LLM extraction prompt output (structured JSON, parsed by ingestion layer):**

```json
{
  "themes": ["<str>"],
  "tickers": [
    {
      "ticker": "<str>",
      "company": "<str>",
      "source": "explicit | inferred",
      "theme": "<str>",
      "rationale": "<str>"
    }
  ]
}
```

## Constraints

- Decomposition method is configurable at runtime via env var
  `BREADWINNER_INGEST_DECOMP=transcript|gemini` — not hardcoded
- If Gemini is used: API key via `GEMINI_API_KEY` env var; never committed
- Thematic analysis always uses Claude via the Anthropic SDK — do not add a second
  LLM SDK solely for analysis
- Market cap threshold ($2B) is configurable via breadwinner config, not hardcoded
- Audit logs are append-only; never overwrite a previous run's log file
- `bw ingest` must not trigger any trade or plan execution — read + watchlist-write only
- No direct imports from breadmin-youtube; queue store is accessed via SQLite path only

## Key Unknowns

1. **[P1]** Transcript extraction vs. Gemini multimodal video analysis: which should be
   the primary decomposition method? Gemini can process a YouTube video URL directly,
   capturing visual signals (slides, charts, on-screen tickers, annotations) that
   text transcripts miss — but it introduces a new API dependency and per-video cost.
   The transcript path is free and already understood (v0.5.0 original design), but
   misses visual context. Research stage must: (a) test Gemini video analysis on 3–5
   representative financial videos and evaluate signal quality vs transcript, (b) price
   the per-video Gemini cost at expected queue volumes, (c) determine whether Gemini
   can accept a YouTube URL directly or requires a downloaded file. The answer
   determines whether `GEMINI_API_KEY` becomes a hard dependency or an optional
   enhancement.

2. **[P1]** Does Alpaca's asset API return market cap data, or is a separate financial
   data source required? If Alpaca does not expose market cap on its asset records,
   breadwinner must integrate a lightweight financial data provider (e.g. yfinance).
   The answer determines whether this milestone introduces a new dependency. Research
   stage must verify Alpaca's asset payload and identify the fallback.

---

## Non-Goals

- Scheduled / cron-based ingestion (breadmin-jobman v0.4.0-signal-grid)
- Non-YouTube sources: podcasts, earnings call transcripts, SEC filings (v0.6.0)
- Automated plan generation triggered by ingestion
- Sentiment scoring or IV-rank filtering at ingestion time
- Speaker attribution or host-vs-guest signal weighting
- TradingView watchlist sync
