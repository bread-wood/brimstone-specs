## Overview

High-stakes portfolio decisions (large position changes, drawdown response, strategy
shifts, ambiguous exit signals) are routed through moot's financial council before
execution. This milestone builds the breadwinner side of the bridge: a deliberation
adapter that formats portfolio proposals for moot, a verdict receiver that translates
council outcomes into breadwinner actions, and an enriched options/crypto state snapshot
that gives financial council agents full portfolio context.

All integration logic avoids direct imports between breadwinner and moot. The
coordination layer lives in breadmin-jobman (see breadmin-jobman v0.2.0-verdict-bridge).
Breadwinner's role is: expose a clean state snapshot, consume verdict signals, and log
the results in its audit trail.

**Prerequisite:** v0.3.0-iron-harvest should complete (full options state must be
available for portfolio context snapshots); breadmin-jobman v0.2.0-verdict-bridge must
be in progress (the contract must be defined before the breadwinner side is built).

## Success Criteria

- [ ] `bw snapshot --format=jobman` outputs a versioned JSON document containing equity
  positions, options positions (with live Greeks), crypto holdings, and analytics
  (Sharpe, drawdown, IV rank for key positions) â€” readable by breadmin-jobman
- [ ] Breadwinner's planning pipeline identifies deliberation-worthy decisions per
  configured thresholds and sets a `requires_deliberation: true` flag on the plan
- [ ] A `breadwinner-council-signal` breadmin handler receives a verdict signal, logs it
  to the audit trail, and sets the plan state to `approved` / `rejected` / `deferred`
- [ ] When a plan is in `pending_deliberation` state, `bw execute-trades` refuses to
  execute and prints a message indicating the plan is awaiting moot verdict
- [ ] When a plan transitions to `approved`, `bw execute-trades` proceeds normally
- [ ] The audit trail records: plan ID, deliberation submitted at, verdict received at,
  verdict outcome, council confidence level
- [ ] `make test && make lint` pass on mainline

## Scope

**Included:**

- `bw snapshot --format=jobman`: enriched portfolio state document (equity, options +
  Greeks, crypto, analytics, IV rank)
- Deliberation trigger evaluation in `services/planning.py`: configurable thresholds
  (net debit/credit size, rebalance %, exit rule firing on large positions, ambiguous LLM signals)
- Plan state machine: `pending_deliberation`, `approved`, `rejected`, `deferred` states
  added to plan lifecycle
- `handlers/breadmin.py`: `breadwinner-council-signal` handler â€” receives verdict from
  breadmin-jobman, updates plan state, logs to audit trail
- Audit trail extension: deliberation fields in DB
- `bw execute-trades` guard: refuses execution when plan is `pending_deliberation`
- `ConnectionSignal` emission: after each planning run, emit structured connection signals
  to jobman's knowledge store capturing the reasoning artifacts (which setups triggered,
  which were passed, IV environment, LLM rationale summary); gives the knowledge graph
  access to financial reasoning without requiring forge skill adoption

**Explicitly excluded:**

- Financial council implementation (moot v0.6.0-trading-floor)
- breadmin-jobman verdict routing (breadmin-jobman v0.2.0-verdict-bridge)
- Direct imports between breadwinner and moot

## Failure Modes

**Deliberation state:**
- Verdict never arrives (moot unavailable) â†’ plan remains in `pending_deliberation`;
  heartbeat surfaces stale pending plans; operator can manually override via CLI
- Verdict arrives with unrecognized schema version â†’ log error; leave plan in
  `pending_deliberation`; alert operator
- Plan state DB write fails after verdict received â†’ retry once; log error; do not
  silently lose the verdict

**Snapshot:**
- Live Greeks unavailable for an options position â†’ include position in snapshot with
  Greeks marked as unavailable; do not omit the position
- Analytics computation fails (e.g. insufficient history for Sharpe) â†’ include what is
  computable; mark unavailable fields explicitly in the snapshot

  "equity_positions": [

## Constraints

- No direct imports from moot anywhere in breadwinner source
- `bw snapshot --format=jobman` schema is versioned; breadwinner must not change the
  schema without bumping `schema_version`
- Deliberation thresholds are configured via env or YAML config, not hardcoded
- The `pending_deliberation` guard in `bw execute-trades` must be bypassable with an
  explicit `--force` flag for emergency overrides (with a loud warning)
