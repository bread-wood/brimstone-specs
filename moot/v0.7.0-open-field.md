# moot v0.7.0 — Open Field

**Theme:** Expanded data integrations — Reddit signals and options/crypto state via
JobMan.

---

## Overview

Two integration expansions that broaden what councils have access to:

1. **Reddit signals**: A JobMan job (in breadmin-jobman) runs Reddit ingestion and
   writes summaries to a signals store. Moot defines the consumption interface — how
   councils query the store and inject signal context into deliberations.

2. **Options/crypto state**: Updated breadwinner integration surfaces options and crypto
   positions to the financial council. The state is pushed from breadwinner via the
   JobMan interface established in v0.6.0.

Both expansions are **data consumption changes in moot** — the actual data collection
(Reddit fetching, breadwinner state extraction) lives in JobMan job handlers, not in
moot integrations. Moot's `src/moot/integrations/` is not the place for data collection;
it is the place for consuming signals that JobMan delivers.

**Prerequisite:** v0.6.0-trading-floor must complete; the verdict handler and JobMan
integration contract must be stable before the options/crypto state layer is extended.

---

## Success Criteria

- [ ] A financial council deliberation includes a "social signals" context block drawn
  from the signals store when available — updated within the last 24 hours
- [ ] The financial council's context block includes open options positions (strike,
  expiry, delta, P&L) when the decision involves an optionable position
- [ ] The financial council's context block includes crypto holdings (coin, quantity,
  current value) when the decision involves a crypto-adjacent portfolio action
- [ ] No Reddit fetching code exists in `moot` source — signal ingestion is a JobMan job
  in breadmin-jobman; moot only reads from the signals store
- [ ] No direct import from `breadwinner` appears in `moot` source — all portfolio state
  is received via the JobMan integration contract
- [ ] When the signals store is stale (> 24 hours since last write) or absent, the
  deliberation proceeds without social signals context; a warning appears in the output
- [ ] `make test && make lint` pass on mainline

---

## Scope

**Included (moot side):**

- Signals store reader: reads Reddit summaries from the store written by the JobMan job
- Council context adapter: injects signals-store contents into financial council
  deliberation context
- Options/crypto context adapter: receives options and crypto state from the JobMan
  integration contract (pushed by breadwinner job), formats as structured council context
- Staleness handling: graceful degradation when signals store is absent or stale

**Out of scope for moot (lives in breadmin-jobman):**

- Reddit fetching, post scoring, sentiment computation — all implemented as a JobMan job
- The JobMan job schedule and trigger configuration
- Breadwinner state extraction into the JobMan integration layer

---

## Failure Modes

**Signals store boundary:**
- Signals store absent → log warning; deliberation proceeds without social signals context
- Signals store stale (> 24 hours) → inject context with a staleness warning visible to
  the council
- Signals store read error → log error; treat as absent; do not crash the deliberation

**Options/crypto state boundary:**
- JobMan integration contract payload absent → deliberation proceeds without portfolio
  context; log warning noting context is unavailable
- Payload malformed or missing expected fields → use only the fields present; log warning;
  do not crash
- Contract version mismatch → log error with version info; refuse the payload rather than
  silently misinterpreting it
- Options/crypto context not relevant to current deliberation topic → do not inject;
  relevance detection is a council-layer concern

## Interface Contracts

**Signals store schema (written by JobMan Reddit job, read by moot):**

```json
{
  "schema_version": "1",
  "as_of": "<iso8601>",
  "subreddits": [
    {
      "name": "<subreddit>",
      "sentiment": "bullish | bearish | neutral",
      "key_themes": ["<theme>"],
      "summary": "<string>",
      "post_count": 25,
      "fetched_at": "<iso8601>"
    }
  ]
}
```

**JobMan → moot portfolio state event (from breadwinner job, defined in v0.6.0):**

```json
{
  "schema_version": "1",
  "as_of": "<iso8601>",
  "options_positions": [
    {
      "ticker": "<string>",
      "type": "CSP | CC | <other>",
      "strike": 0.0,
      "expiry": "<YYYY-MM-DD>",
      "delta": 0.0,
      "pnl_pct": 0.0
    }
  ],
  "crypto_holdings": [
    { "coin": "<string>", "quantity": 0.0, "value_usd": 0.0 }
  ]
}
```

## Constraints

- No Reddit fetching code in `moot` — signals store is the only interface
- No direct imports from `breadwinner` in `moot` source
- Signals store is owned by breadmin-jobman (SQLite in jobman's data dir); moot
  reads it via the path configured in `JOBMAN_SIGNALS_DB` — moot never writes to it
- All new code is type-annotated, async-compatible, and passes `ruff check`
- Integration contracts are versioned (`schema_version`); moot must reject payloads
  with unsupported schema versions

## Key Unknowns

1. **[P1]** Reddit API access model: should the JobMan Reddit job use PRAW, a scraper,
   or a breadmin-level abstraction? This determines the implementation of the JobMan job
   (breadmin-jobman scope, not moot scope), but the answer affects what fields moot can
   expect in the signals store. Research stage must answer this and define the signals
   store schema before moot's consumption layer is designed.

2. **[P1]** How should options/crypto state be structured for moot consumption? What
   fields does a financial council agent need from breadwinner's options/crypto state, and
   at what granularity? The answer determines the signals schema that moot's context
   adapter reads. Research stage must answer this before the context adapter is designed.

---

## Non-Goals

- Reddit fetching, post scoring, sentiment computation (JobMan job, not moot)
- JobMan job scheduling or trigger configuration
- Real-time Reddit streaming
- Sentiment analysis of Reddit comments (top posts only)
- Twitter/X, Discord, or other social platform ingestion
- Automated trade execution from signals (moot signals; breadwinner executes)
- breadministrator implementation
