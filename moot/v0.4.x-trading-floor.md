# moot v0.4.x — Trading Floor

> **Retrospective spec** — describes what was built, not what to build.
> The pipeline has already run for this version.

**Theme:** Adds a financial council to moot's deliberation engine.

## Overview

Extends moot with a financial council purpose-built for portfolio decision deliberation.
The council follows the PRISM procedure and is staffed by financial agent personas defined
in YAML. High-stakes financial decisions (large position changes, strategy shifts) trigger
a deep pass through the full council before breadwinner acts. Includes heartbeat checks
for stale and urgent docket issues, and a roster update replacing Rampart with Meridian.

---

## What Was Built

| Area | Detail |
|------|--------|
| Financial council manifest | YAML config defining council membership and PRISM procedure |
| Agent personas | `financial_analyst`, `risk_officer`, `macro_strategist`, `sector_specialist`, `quant_researcher` |
| Domain reference classes | `FinancialDecision`, `PortfolioAction`, `RiskEvent` |
| Deep pass triggers | Configurable thresholds: position size, drawdown %, strategy change flag |
| Verdict handler | Structured outcome: `approve` / `reject` / `defer` with rationale |
| Heartbeat | Checks for docket issues stale > N days or marked urgent |
| Roster update | Rampart removed; Meridian (systems integrator) added |

## Non-Goals (deferred to v0.5.x)

- Options and crypto state updates in portfolio context
