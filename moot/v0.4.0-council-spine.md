# moot v0.4.0 — Council Spine

**Theme:** Establish the `CouncilPlugin` protocol and restructure the package around it.

---

## Overview

Moot has grown to ~15,600 lines with council logic (Prism, Inquiry, future RGB) tangled
into shared infrastructure (identity, debate, routing, Mirror). This milestone defines a
`CouncilPlugin` protocol — the interface contract all councils must satisfy — restructures
`src/moot/` so each council is an isolated plugin, removes stale OAuth/Fitbit stubs and
the Rampart agent, and refines the CLAUDE.md module table so brimstone can dispatch
isolated agents per module.

No new council is built in this milestone. Existing councils (Prism, Inquiry) are migrated
to the new interface.

## Failure Modes

**Migration failures:**
- Prism or Inquiry fails to satisfy `CouncilPlugin` → block merge; do not ship partial protocol
  compliance
- `procedures/` merge breaks routing behavior → write regression tests for routing paths
  before deleting the directory
- Stale import in an unexpected file after stub removal → ruff and mypy will catch; fix
  before merge

**Structural failures:**
- `CouncilPlugin` interface too broad → councils pass the check but the interface is
  meaningless; Key Unknown #1 research must answer this before design begins
- Module table granularity wrong → revise after impl if agent isolation proves
  too coarse or too fine in practice

## Interface Contracts

**`CouncilPlugin` protocol — fully resolved:**

```python
from __future__ import annotations
from dataclasses import dataclass, field
from typing import Any, Protocol

@dataclass
class Topic:
    topic_id: str
    text: str
    domain: str = ""
    context: dict[str, Any] = field(default_factory=dict)

@dataclass
class CouncilOutput:
    council_id: str
    verdict: str          # "approve" | "reject" | "defer" | free-text for non-financial
    rationale: str
    confidence: str       # "high" | "medium" | "low"
    dissents: list[str] = field(default_factory=list)  # ["agent_id: brief position"]

class CouncilPlugin(Protocol):
    council_id: str       # class-level attribute, unique per council

    async def run(self, topic: Topic) -> CouncilOutput: ...
```

No lifecycle hooks in v0.4.0. `init`/`session_start`/`session_end` are deferred to v0.5.0
when the RGB council's identity model requires them. Prism and Inquiry both satisfy this
protocol with their existing `run()` signatures — migration is a rename + type annotation,
not a behavioral change.

**Router decision — resolved: stay on Claude Haiku:**

The GPT-5 Nano router is replaced with `claude-haiku-4-5-20251001` via `breadmin-llm`'s
`ProviderRegistry`. Rationale: GPT-5 Nano adds an external OpenAI dependency that
breadmin-llm already abstracts; Claude Haiku is already registered, costs are equivalent,
and the routing prompt is trivial classification that Haiku handles well. No behavior
changes to routing logic — only the model identifier changes.
