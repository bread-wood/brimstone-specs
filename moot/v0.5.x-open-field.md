# moot v0.5.x — Open Field

**Theme:** Broadens moot's data inputs and integration surface.

## Overview

Broadens the information available to moot's deliberation agents. Reddit integration adds
social signal ingestion — subreddit summaries and sentiment aggregates are injected as
council context. The portfolio integration layer is extended to surface options and crypto
positions to the financial council. All moot↔breadwinner coordination is routed through
breadministrator, keeping the two systems architecturally independent.

---

## Success Criteria

- [ ] A moot deliberation on a financial issue includes a "social signals" context block
  drawn from configured subreddit summaries updated within the last 24 hours
- [ ] `moot ingest reddit --subreddit <name>` fetches the top posts from the subreddit,
  produces a sentiment summary, and stores it in the signals store
- [ ] The financial council's context block includes open options positions (strike, expiry,
  delta, P&L) when the decision involves an optionable position
- [ ] The financial council's context block includes crypto holdings (coin, quantity,
  current value) when the decision involves a crypto-adjacent portfolio action
- [ ] No direct import from `breadwinner` appears in `moot` source — all portfolio state
  is received via a breadministrator integration contract (event or API call)
- [ ] `make test` passes clean after all changes

---

## Scope

**Included:**

- **Reddit ingestion**: fetch top posts from configured subreddits via Reddit API;
  produce a sentiment summary (bullish / bearish / neutral + key themes); store in signals
  store; inject into deliberation context
- **Options state context**: receive options positions from breadministrator; format as
  structured context for the financial council
- **Crypto state context**: receive crypto holdings from breadministrator; format as
  structured context for the financial council
- **breadministrator integration contract**: define the event/API contract that
  breadministrator uses to push portfolio state to moot; implement the moot receiver side

---

## Failure Modes

**Reddit API boundary:**
- Reddit API unavailable → log warning to stderr, continue deliberation without social
  signals context; do not block the deliberation
- Reddit API rate limit → exponential backoff up to 3 retries; if exhausted, skip social
  signals for this deliberation with a warning
- Reddit API auth failure → log error, disable Reddit ingestion for the session; alert
  operator via Telegram if configured
- Subreddit not found or private → log warning, skip that subreddit; do not fail the
  entire ingestion run

**breadministrator integration boundary:**
- breadministrator unreachable when moot requests portfolio state → proceed with
  deliberation without portfolio context; log warning noting that context is stale
- Portfolio state payload malformed or missing expected fields → log warning, use only
  the fields present; do not crash the deliberation
- Contract version mismatch → log error with version info; refuse to proceed with
  incompatible payload rather than silently misinterpreting it

**Signals store:**
- Signals store write fails → log error, continue in-memory; stale signals persist until
  next successful write
- Signals store data stale (>24 hours since last Reddit ingest) → inject context with a
  staleness warning visible to the council

## Interface Contracts

**CLI commands (new in v0.5.x):**

```
moot ingest reddit --subreddit <name> [--limit <N>]
```

| Stream | Content |
|--------|---------|
| stdout | Summary of ingested posts and computed sentiment |
| stderr | Warnings and errors |
| exit 0 | Ingestion succeeded |
| exit 1 | Auth failure, subreddit not found, or unrecoverable API error |

**breadministrator → moot portfolio state event:**

```json
{
  "schema_version": "1",
  "as_of": "<iso8601>",
  "options_positions": [
    {
      "ticker": "AAPL",
      "type": "CSP",
      "strike": 170.0,
      "expiry": "2025-03-21",
      "delta": -0.20,
      "pnl_pct": 0.35
    }
  ],
  "crypto_holdings": [
    { "coin": "BTC", "quantity": 0.5, "value_usd": 32500.0 }
  ]
}
```

**Environment variables (new in v0.5.x):**

| Variable | Required | Purpose |
|----------|----------|---------|
| `REDDIT_CLIENT_ID` | Yes (for Reddit ingestion) | Reddit API OAuth client ID |
| `REDDIT_CLIENT_SECRET` | Yes (for Reddit ingestion) | Reddit API OAuth client secret |
| `REDDIT_USER_AGENT` | No | User-agent string for Reddit API requests |

## Constraints

- No direct imports from `breadwinner` in `moot` source — all portfolio state received
  via the breadministrator contract
- Reddit integration must degrade gracefully — deliberations proceed without social signals
  if Reddit is unavailable
- Signals store uses SQLite (consistent with existing moot persistence layer)
- All new code is type-annotated, async-compatible, and passes `ruff check`
- The breadministrator integration contract is versioned (`schema_version` field);
  moot must reject payloads with unsupported schema versions

---

## Key Unknowns

1. **[P1]** What is the correct breadministrator integration mechanism — push (breadministrator
   calls a moot API endpoint when portfolio state changes) vs. pull (moot queries
   breadministrator when a deliberation begins)? The answer determines whether moot needs
   an inbound HTTP endpoint, a shared event bus, or a polling mechanism. This architectural
   decision must be resolved before the integration layer is designed.

2. **[P2]** Reddit sentiment computation: should the sentiment summary be computed by a
   lightweight heuristic (keyword scoring on post titles + scores) or by calling an LLM
   on each post? The answer affects latency, cost, and accuracy of the social signals context.
   A heuristic is cheaper and faster; an LLM produces higher-quality summaries but adds
   per-ingestion cost.

---

## Non-Goals

- Direct moot ↔ breadwinner imports (all coordination through breadministrator)
- Real-time Reddit streaming (ingestion is periodic, not event-driven)
- Sentiment analysis of Reddit comments (top posts only in this version)
- Twitter/X, Discord, or other social platform ingestion
- The breadministrator implementation itself (out of moot's scope)
- Options order execution triggered by moot deliberation verdicts
