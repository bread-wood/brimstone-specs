# moot v0.5.0 — RGB Council

**Theme:** Build the RGB deliberation council (Blue/Green/Red + Grey facilitator).

---

## Overview

The RGB council is the flagship deliberation engine for life decisions. Three agents —
Blue (ambition), Green (cultivation), Red (limits) — debate in structured rounds.
Grey, a neutral process facilitator, detects convergence or divergence and produces
the final output. This milestone builds the RGB council as a `CouncilPlugin` and
integrates it with Mirror under therapeutic isolation.

**Prerequisite:** v0.4.0-council-spine must complete; RGB is implemented as a
`CouncilPlugin` from the start.

**Source of truth:** Full spec exists at `docs/specs/moot-v0.6.0-rgb-spec.md`.
That document is canonical for success criteria, failure modes, interface contracts,
and constraints. This file is the brimstone milestone descriptor; refer to the full
spec for implementation details.

---

## Success Criteria

See `docs/specs/moot-v0.6.0-rgb-spec.md` for the full testable criteria. Summary:

- [ ] Blue, Green, Red agents produce role-authentic Round 1 outputs for any given topic
- [ ] Each agent's Round 2 references specific claims from both other agents' Round 1
- [ ] Grey terminates deliberation after at most 3 rounds
- [ ] Grey produces a convergence synthesis or divergence report with alignment pattern
  diagnostics per the canonical spec
- [ ] Grey never contributes content to the deliberation
- [ ] Agent identity (worldview state, journal, relationship map, track record) persists
  across sessions
- [ ] Mirror can interview each RGB agent under therapeutic isolation
- [ ] RGB council satisfies `CouncilPlugin` protocol
- [ ] `make test && make lint` pass on mainline

---

## Scope

**Included:**

- Blue, Green, Red deliberation agents
- Grey facilitator: process management, convergence/divergence detection, output synthesis
- Three-round deliberation structure with early termination
- Convergence synthesis and divergence report output formats
- Alignment pattern diagnostics
- Six-layer agent identity model (personality seed, worldview state, journal,
  relationship map, track record, context window)
- Mirror integration under therapeutic isolation
- `CouncilPlugin` compliance

**Explicitly excluded:**

- Multiple agents per color
- Scheduling or automated triggering
- Routing architecture changes
- Grey participating in deliberation content
- Dual-identity or infiltrator mechanics
- Arbiter, Black/White configurations (retired)
- Personality seed definitions (deferred to research — Key Unknown #1)

---

## Key Unknowns

1. **[P1]** What personality seeds produce authentic Blue/Green/Red reasoning
   (temperament) rather than role-playing a label? This is the same Key Unknown from the
   canonical spec. Research stage must answer: what does each color sound like in
   conversation, how do blind spots manifest naturally, and how should seeds be structured
   so the orientation is intrinsic rather than instructed.

2. **[P1]** What heuristics should Grey use to detect convergence vs divergence vs
   productive-but-unresolved? Grey needs to assess whether positions are actually moving
   toward each other or just softening language without changing substance. Naive keyword
   matching will fail. Research stage must answer before Grey's convergence algorithm
   is designed.
