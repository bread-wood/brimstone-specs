# breadforge v0.2.0 — Phase 2

**Theme:** Production hardening — cross-repo blocking, design stage, credential proxy, GHA, multi-backend.

---

## Overview

breadforge v0.1.0 shipped a working DAG executor: plan → build → merge → readme, with
repair agents on CI failure and crash recovery. Phase 2 adds the features that make it
production-grade and complete the pipeline:

- **Cross-repo wait nodes** (2B): milestones blocked by foreign-repo releases dispatch
  a wait node instead of proceeding immediately
- **Design stage** (2F): plan nodes can emit design nodes before build nodes, producing
  HLD/LLD artifacts that build agents consume instead of architecting on the fly
- **Credential proxy** (2A): raw API keys are never in subprocess env; a loopback proxy
  holds the real key and issues scoped session tokens per agent
- **GHA deployment** (2C): breadforge runs unattended in CI, triggered by GitHub issues
  labeled `pipeline`, without a local Claude session
- **Multi-backend** (2D): research and plan nodes can use Gemini or GPT-4.1; build nodes
  always use Claude Code
- **Cross-model consensus** (2E): P0 unknowns run through two backends; contradictions
  are flagged `[DISPUTED]` before plan-refine sees the findings

## Success Criteria

- [ ] A milestone with `milestone_blocked_by` in its CampaignBead emits a `wait` node
  that polls `gh release view`; build dispatch holds until the release appears
- [ ] A spec with `novel-domain`, `security`, or `multi-module-coordination` risk flag
  produces a design PR before any build nodes dispatch; build agents receive
  `design_lld_path` in context
- [ ] `ANTHROPIC_API_KEY` does not appear in any agent subprocess environment; health
  check confirms proxy running
- [ ] `breadforge run` triggered by a GitHub Actions `issues` event (label `pipeline`)
  completes without a local Claude session
- [ ] `BREADFORGE_RESEARCH_MODEL=gemini-2.0-flash` causes research nodes to use Gemini;
  runner log shows `backend=gemini`
- [ ] P0 Key Unknown research runs through two backends; consensus node marks
  contradictions `[DISPUTED]`; plan-refine node uses annotated output
- [ ] `make test && make lint` pass on mainline

## Scope

**2B — Cross-Repo Wait Nodes:**
- New node type `"wait"` in `NodeType`
- `WaitHandler`: polls `gh release view {tag} --repo {repo}` every 60s; resolves when release exists
- `graph/builder.py`: before plan node, check `CampaignBead.milestone_blocked_by`; emit wait node(s) with plan depending on them
- Status display: wait nodes show `waiting for {repo}@{tag}`

**2F — Research → Design → Impl Pipeline:**
- New node type `"design"` in `NodeType`
- `DesignHandler`: runs Claude Sonnet agent with Read/Write/Glob/Bash; agent writes HLD at `docs/design/{milestone}/hld.md` and per-module LLDs at `docs/design/{milestone}/lld-{module}.md`; opens a design PR; stores artifact paths in node output
- `plan.py` expansion logic:
  - `confidence < 0.6 and unknowns` → research → plan-refine → design → build
  - `risk_flags` contains `novel-domain | security | multi-module-coordination` → design → build
  - Otherwise → build directly
- Build node context: `design_hld_path`, `design_lld_path` passed to build agent prompt
- Design artifacts stored at `~/.breadforge/beads/{owner}/{repo}/design/{node-id}/`

**2A — Credential Proxy:**
- `agents/proxy.py`: loopback HTTP server (`localhost:{port}`); intercepts Anthropic base URL; holds real key in memory; issues scoped session tokens per agent; revokes on node completion
- `agents/runner.py`: `_build_env()` injects `ANTHROPIC_BASE_URL=http://localhost:{port}` + session token instead of raw key
- `config.py`: `proxy_port: int = 9473`
- `health.py`: proxy health check

**2C — GHA Deployment:**
- `.github/workflows/breadforge.yml`: triggered on `issues` event (label `pipeline`); ephemeral runner; reads `BREADFORGE_CONFIG` secret
- `cli.py`: `--ci` flag disables Rich live display, emits plain log output
- `health.py`: `check_gh_token()`, `check_no_tty()` for CI context

**2D — Multi-Backend:**
- `agents/backend.py`: `Backend` protocol; `ClaudeBackend` (wraps run_agent subprocess); `GeminiBackend` / `OpenAIBackend` (SDK direct; research + plan nodes only); `BackendRegistry` maps model name → backend
- Build nodes always use `ClaudeBackend`
- `config.py`: `research_model`, `plan_model` config fields (override per node type)

**2E — Cross-Model Consensus:**
- New node type `"consensus"` in `NodeType`
- `ConsensusHandler`: reads two research findings docs; calls LLM to diff; marks disagreements `[DISPUTED]`; writes merged findings
- `plan.py`: for P0 unknowns, emit two research nodes (one per backend) + one consensus node; plan-refine depends on consensus node

## Failure Modes

**Wait node:**
- Release never appears → wait node abandons after `max_retries` (configurable, default 1440 = 24h at 60s intervals)
- `gh release view` call fails → retry; log warning; do not abandon on transient failure

**Design node:**
- Agent fails to produce HLD → retry up to `max_retries=3`; if exhausted, build nodes proceed without design context (logged warning); do not block the pipeline
- Design PR has merge conflicts → design node marked abandoned; build nodes proceed without design

**Credential proxy:**
- Proxy fails to start → health check fails; `breadforge run` exits with error
- Token revocation fails → log error; do not retry; accept the leak

**GHA:**
- `BREADFORGE_CONFIG` secret missing → exit 1 with clear error
- Runner runs out of time → merge nodes retry on next trigger

## Interface Contracts

**Wait node context:**
```json
{ "wait_repo": "owner/repo", "wait_tag": "v0.6.0", "poll_interval_seconds": 60 }
```

**Design node context:**
```json
{
  "milestone": "v0.4.0",
  "spec_file": "/path/to/spec.md",
  "repo": "owner/repo",
  "repo_local_path": "/tmp/clone",
  "plan_artifact": { "...": "..." },
  "research_node_ids": ["..."]
}
```

**Design node output:**
```json
{
  "hld_path": "docs/design/v0.4.0/hld.md",
  "lld_paths": { "module_name": "docs/design/v0.4.0/lld-module_name.md" },
  "pr_number": 42,
  "branch": "design/v0.4.0"
}
```

**Build node context additions (from design):**
```json
{ "design_hld_path": "docs/design/v0.4.0/hld.md", "design_lld_path": "docs/design/v0.4.0/lld-my_module.md" }
```

## Constraints

- 2B and 2F have no external dependencies — ship first
- 2A is prerequisite for 2C and 2D
- Build nodes always use `ClaudeBackend` (Claude Code subprocess) — non-Claude backends for research/plan only
- Design PR is squash-merged by the design node itself (not a separate merge node) — design is not user-facing code, it's an artifact
- All new handlers registered in `make_handlers()`
- `NodeType` literal updated atomically with handler registration

## Key Unknowns

1. **[P2]** GHA ephemeral runner: does `claude --print` work without a TTY? Need to
   verify `claude` CLI behavior in non-interactive CI environment before 2C design.
   Workaround: `--ci` flag + `CLAUDE_CODE_NON_INTERACTIVE=1` env var.

2. **[P2]** Design PR merge strategy: should design PRs be squash-merged by the design
   node, or should a separate merge node handle them? Squash-merge in design node
   is simpler but skips CI; a merge node adds CI coverage on design docs. Decision
   deferred to design stage.

---

## Non-Goals

- Personality seeds or identity models for design agents
- Real-time streaming of agent output (all polling-based)
- Parallel design nodes per module (one design node covers all modules in one PR)
- Vision or image inputs to any handler
