# Minesweeper v0.6.x — Challenge

**Theme:** A reason to return. Skill made visible over time.

## Overview

Adds a daily challenge mode and a board difficulty rating system. The daily
challenge presents the same board to every player on the same calendar day
(derived from a deterministic date seed); completing it extends a streak.
The difficulty rater analyses any board configuration and estimates how likely
an optimal player is to win — giving players a calibrated sense of what they
are attempting before they start. Together these features transform Minesweeper
from a session game into a daily practice.

---

## Success Criteria

- [ ] `minesweeper daily` starts today's daily challenge; the board is identical
  for all players on the same date regardless of system timezone
- [ ] `minesweeper daily` on the same date always produces the same board; on a
  different date always produces a different board
- [ ] Completing `minesweeper daily` prints `Daily challenge complete! Streak: N`
  and records the result; losing prints `Daily challenge lost. Streak reset.`
- [ ] Playing `minesweeper daily` more than once on the same day after winning
  prints `Already completed today's challenge. Streak: N` and exits 0
- [ ] `minesweeper rate --width 9 --height 9 --mines 10` prints a difficulty
  rating: estimated win rate for an optimal player using the v0.2.x solver with
  probabilistic guessing
- [ ] `minesweeper rate --width 30 --height 16 --mines 99` prints a lower
  estimated win rate than beginner; the ordering matches known empirical win rates
- [ ] `minesweeper daily --preview` prints today's board config (dimensions,
  mines, difficulty rating) without starting the game
- [ ] `minesweeper streak` prints current daily streak and longest streak
- [ ] `make test` passes clean

---

## Scope

**Included:**

- Daily challenge: board seed derived from `YYYY-MM-DD` in UTC; same seed for
  all players globally on the same date
- Streak tracking: consecutive calendar days with a completed (win) daily challenge;
  stored in the stats file from v0.4.x
- Completion guard: playing the daily a second time after a win on the same day
  is a no-op with a message; after a loss the board is not re-offered (one attempt per day)
- `minesweeper rate [--width N] [--height N] [--mines N]`: run the v0.2.x
  auto-solver N times (configurable, default 200) on randomly generated boards
  with the given config; report mean win rate and standard error
- `--preview`: print the daily challenge config and difficulty estimate without starting
- `minesweeper streak`: print streak summary from stats file

---

## Failure Modes

**Daily:**
- Stats file not writable → daily challenge plays normally; completion not recorded;
  `warning: cannot record daily result: <reason>` to stderr at game end
- System clock substantially wrong (detected via UTC date mismatch) → use UTC date;
  print `warning: system clock may be incorrect; using UTC date`

**Rating:**
- `--mines` ≥ total cells → `error: invalid configuration` to stderr, exit 1
- Rating simulation takes > 30 s → print partial result with note `(partial: N samples)`

## Interface Contracts

**CLI additions:**

```
minesweeper daily [--preview]
minesweeper rate [--width N] [--height N] [--mines N] [--samples N]
minesweeper streak
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--preview` | off | Print daily config without starting |
| `--samples` | `200` | Number of simulation runs for difficulty rating |

**`minesweeper rate` output (stdout):**

```
Board: 9×9, 10 mines
Simulated win rate: 87.4% ± 2.1%  (200 samples, optimal solver)
Difficulty: Easy
```

**`minesweeper daily --preview` output:**

```
Today's challenge (2026-03-03 UTC)
Board: 16×16, 40 mines
Estimated win rate: 54.2%
Difficulty: Medium
```

**Difficulty tiers (derived from estimated win rate):**

| Win rate | Label |
|----------|-------|
| ≥ 80% | Easy |
| 50–79% | Medium |
| 25–49% | Hard |
| < 25% | Expert |

## Constraints

- Depends on v0.2.x solver for the difficulty rating simulation and
  v0.4.x stats store for streak persistence
- The daily seed derivation must be deterministic, timezone-independent, and
  produce visually distinct boards on consecutive days (no streaks of similar boards)
- The `rate` command runs the solver N times in-process; no subprocess spawning
- Streak tracking uses calendar dates in UTC; the day boundary is midnight UTC

---

## Key Unknowns

1. **[P1]** Is a simulated win rate (run solver N times, count wins) a sufficient
   and stable difficulty signal — or does high variance in outcomes across random
   board instances of the same configuration make the estimate too noisy to be
   meaningful without thousands of samples? The answer determines the default
   sample count and whether the difficulty rating is a useful feature or an
   unreliable number.

2. **[P2]** Should the daily challenge use a no-guess board (guaranteed solvable)
   or a standard board (may require guessing) — and does the choice significantly
   affect player experience given that most players will attempt the daily once and
   accept the outcome? The answer determines whether the daily challenge integrates
   with v0.3.x no-guess generation and whether losing to a forced guess feels fair.

---

## Non-Goals

- Server-side daily challenge (seed is locally computed; no network required)
- Shared leaderboards for daily challenge times
- Retrospective daily challenges (only today's challenge available)
- Custom daily challenge schedules
- Windows support
