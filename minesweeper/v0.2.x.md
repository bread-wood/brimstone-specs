# Minesweeper v0.2.x — Solver

**Theme:** The computer plays too. Certainty extracted from constraints.

## Overview

Adds a constraint-based solver that identifies provably safe and provably mined
cells from the current board state. Exposed as two interfaces: a hint system
(press `h` to highlight one safe move) and an auto-solve mode that plays the
game to completion or deadlock. When no certain move exists, the solver reports
the cell with the lowest mine probability. This version does not generate
no-guess boards — it solves whatever board it is given, including boards that
eventually require a guess.

---

## Success Criteria

- [ ] In a live game, pressing `h` highlights a provably safe cell; if none
  exists, it highlights the cell with the lowest calculated mine probability
- [ ] `minesweeper --auto-solve --seed <N>` plays a game to completion or
  deadlock without human input and exits 0 with a result report
- [ ] The result report includes: outcome (win/loss/deadlock), moves made,
  cells revealed by flood fill, guesses required, and whether a guess caused a loss
- [ ] `minesweeper --auto-solve --seed <N>` run 1000 times on beginner
  produces a win rate above 85% (known optimal for beginner-difficulty boards)
- [ ] The solver identifies all provably safe cells from a given board state
  in under 100 ms for boards up to 30×30
- [ ] A cell marked safe by the solver is never a mine (zero false safe
  predictions across the full test suite)
- [ ] `make test` passes clean with no live terminal

---

## Scope

**Included:**

- Constraint propagation: for each revealed numbered cell, derive constraints
  on its unrevealed neighbors; propagate until no new safe/mine cells can be
  determined
- Subset reasoning: when one constraint's neighbor set is a subset of another's,
  derive additional constraints
- Probability estimation: for cells where no certain determination is possible,
  compute mine probability from the remaining constraint system
- Hint mode: `h` key highlights one safe cell (lowest probability if no certain
  move); repeated `h` cycles through safe candidates
- Auto-solve mode: `--auto-solve [--seed N]` plays the game without human input;
  guesses at lowest-probability cell when stuck
- Result report: printed to stdout on game end in auto-solve mode

---

## Failure Modes

**Solver:**
- Board state is logically contradictory (should not occur in a valid game) →
  log warning; solver returns no recommendations; game continues
- Solver times out (> 500 ms for boards ≤ 30×30) → hint mode falls back to
  a random unrevealed unflagged cell; logs warning

**Auto-solve:**
- `--seed` not provided → game uses a random seed; seed is printed to stdout
  at start so runs can be reproduced

## Interface Contracts

**CLI additions:**

```
minesweeper [all v0.1.x flags] [--auto-solve] [--seed N]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--auto-solve` | off | Play without human input; print result report on exit |
| `--seed N` | random | Board seed for reproducibility |

**Additional keyboard binding:**

| Key | Action |
|-----|--------|
| `h` | Show hint (highlight safe cell or lowest-probability cell) |

**Auto-solve result report (stdout):**

```json
{
  "seed": 42,
  "outcome": "win",
  "moves": 47,
  "flood_reveals": 120,
  "guesses": 1,
  "guess_caused_loss": false
}
```

| Stream | Content |
|--------|---------|
| stdout | `You win` / `Game over` (interactive) or result report JSON (auto-solve) |
| stderr | `error: <description>` on invalid configuration |
| exit 0 | Game ended or auto-solve completed |
| exit 1 | Invalid configuration |

## Constraints

- Depends on v0.1.x board module; solver reads board state through the existing
  board interface without modifying it
- The solver must be callable as a pure function: `solve(board_state) → (safe_cells, mined_cells, probabilities)`; no side effects
- The solver must be fully testable without a terminal: given a board state,
  assert which cells it marks safe or mined
- Constraint propagation must be correct: a cell marked safe must never be a mine

---

## Key Unknowns

1. **[P1]** Is constraint propagation with subset reasoning sufficient to solve
   all logically solvable positions — or are there board positions that a human
   expert could solve through pattern recognition but that constraint propagation
   cannot reach, requiring a more powerful inference mechanism? The answer
   determines whether the solver is complete for solvable boards or only
   a practical approximation.

2. **[P2]** Does mine probability estimation from the constraint system produce
   estimates accurate enough to measurably improve auto-solve win rates beyond
   random cell selection — specifically, does the lowest-probability cell
   actually survive at a higher rate than a randomly chosen cell across the
   full board space? The answer determines whether probability estimation is
   a meaningful feature or a UI affordance with negligible practical impact.

---

## Non-Goals

- Guaranteed no-guess board generation — v0.3.x
- Optimal guessing strategy (probability is a heuristic; not a proof of optimality)
- Multi-step lookahead (the solver does not simulate future states)
- Exporting solver analysis to a file
- Windows support
