# Minesweeper v0.4.x â€” Records

**Theme:** Every game remembered. Progress visible over time.

## Overview

Adds persistent statistics and game replay. Every completed game is recorded:
outcome, board configuration, duration, move sequence. Statistics are aggregated
per difficulty and viewable at any time. Replays can be played back move-by-move
at configurable speed. The player can finally see whether they are improving.

## Scope

**Included:**

- Game recording: at the end of every game, write a record to the stats store
  containing: board config, seed, outcome, duration, move sequence
- Stats store: append-only file in `~/.minesweeper/stats.jsonl`; one JSON
  record per line
- `minesweeper stats`: aggregated statistics display â€” games played, win rate,
  best time, average time, current win streak; grouped by difficulty
- `minesweeper replay [--last | <file>]`: render game move by move from a
  saved record
- Replay controls: Space to pause/resume, arrow keys to step, `q` to quit
- Replay speed: `--speed 0.5x`, `1x` (default), `2x`, `5x`, `instant`
- `--save <file>`: export a game record to a portable file
- `stats --reset`: clear stats store with confirmation

## Failure Modes

**Stats store:**
- Store file corrupt or unreadable â†’ `minesweeper stats` prints
  `warning: stats file corrupt; showing partial results` and displays whatever
  records are intact
- Store directory not writable â†’ game runs normally; stats are not recorded;
  prints `warning: cannot write stats: <reason>` to stderr at game end

**Replay:**
- Replay file not found â†’ `error: replay file not found: <path>` to stderr, exit 1
- Replay file corrupt â†’ `error: replay file corrupt: <path>` to stderr, exit 1
- Replay board config incompatible with current version â†’ `error: replay version unsupported` to stderr, exit 1

## Constraints

- Depends on v0.1.x for the move representation; every player action must
  be capturable as a typed event (reveal, flag, unflag)
- Stats store is append-only during normal operation; `--reset` is the only
  destructive operation and requires confirmation
- The record format is versioned; unrecognised versions are skipped during
  aggregation, not rejected
- Replay must use the same renderer as the live game; no separate replay renderer
