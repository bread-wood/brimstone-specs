# Minesweeper v0.1.x — Foundation

**Theme:** The classic game, in the terminal, fully playable.

## Overview

A terminal-based Minesweeper implementation with configurable board dimensions
and mine count. Players navigate a grid, reveal cells, and flag suspected mines.
Revealing all safe cells wins; revealing a mine loses. Targets developers who
want a fully-exercised pipeline proof-of-concept with meaningful parallel modules
(board, game logic, renderer each independently testable). The first version that
can be played from end to end.

---

## Success Criteria

- [ ] `minesweeper` starts a 9×9 game with 10 mines, renders a full board, and
  exits 0 on clean quit
- [ ] `minesweeper --width 16 --height 16 --mines 40` starts a 16×16 game with
  40 mines; the rendered board has exactly 16 rows and 16 columns
- [ ] Revealing a cell with 0 adjacent mines automatically reveals all connected
  zero-neighbor cells and their numbered neighbors (flood fill)
- [ ] Revealing a mine ends the game, reveals all mines on the board, and prints
  `Game over` to stdout followed by a newline
- [ ] Revealing the last safe cell wins the game, reveals the full board, and
  prints `You win` to stdout followed by a newline
- [ ] The first cell revealed is never a mine; mine placement is deferred until
  after the first move
- [ ] Flagging a cell marks it; unflagging removes the mark; flagged cells cannot
  be revealed without unflagging first
- [ ] The mine counter (total mines minus flagged cells) is displayed and updates
  on each flag/unflag
- [ ] `minesweeper --mines 81 --width 9 --height 9` prints
  `error: mine count must be less than cell count` to stderr and exits 1
- [ ] `make test` passes clean on macOS and Linux without a live terminal
  (game logic fully testable in isolation)

---

## Scope

**Included:**

- Single-player terminal game loop with interactive keyboard input
- Configurable board: `--width`, `--height`, `--mines`; named presets:
  `--beginner` (9×9, 10), `--intermediate` (16×16, 40), `--expert` (30×16, 99)
- Board rendering: unrevealed cells, revealed cells (digit or blank), flagged
  cells, mine cells (on loss)
- Flood-fill reveal on zero-neighbor cells
- First-move safety: mine placement deferred until after first reveal
- Win and loss detection with full board reveal on game end
- Mine counter and elapsed timer displayed during play
- Clean terminal state on exit (cursor restored, input mode reset)

---

## Failure Modes

**Configuration:**
- `--mines` ≥ total cells → `error: mine count must be less than cell count` to stderr, exit 1
- `--mines < 1` → `error: mine count must be at least 1` to stderr, exit 1
- `--width` or `--height < 2` → `error: board dimensions must be at least 2` to stderr, exit 1
- Unknown flag → `error: unknown option: <flag>` to stderr, exit 1

**Runtime:**
- Terminal too small to render the board → `error: terminal too small: need at least <W>×<H>` to stderr, exit 1
- Terminal resize mid-game → re-render at new size if possible; if still too small, pause and display resize prompt

## Interface Contracts

**CLI:**

```
minesweeper [--width N] [--height N] [--mines N]
            [--beginner | --intermediate | --expert]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--width` | `9` | Board width in cells |
| `--height` | `9` | Board height in cells |
| `--mines` | `10` | Number of mines |
| `--beginner` | — | Preset: 9×9, 10 mines |
| `--intermediate` | — | Preset: 16×16, 40 mines |
| `--expert` | — | Preset: 30×16, 99 mines |

**Keyboard bindings:**

| Key | Action |
|-----|--------|
| Arrow keys / hjkl | Move cursor |
| Enter / Space | Reveal cell |
| `f` | Flag / unflag cell |
| `q` | Quit |

| Stream | Content |
|--------|---------|
| stdout | `You win` or `Game over` on game end; empty otherwise |
| stderr | `error: <description>` on invalid configuration |
| exit 0 | Clean quit or game end |
| exit 1 | Invalid configuration |

## Constraints

- Runs on macOS and Linux; no dependencies beyond the language standard library
  plus at most one terminal rendering library
- No GUI framework — terminal only
- Board logic (state, mine placement, flood fill, win/loss detection) must be
  fully unit-testable without a terminal present
- The board state representation must extend to non-square topologies
  (hexagonal, toroidal) in future versions without a full rewrite of the
  board module
- Minimum board: 2×2 with 1 mine; maximum: 100×100

---

## Key Unknowns

1. **[P1]** What board state representation — 2D array of cell objects, parallel
   flat arrays, coordinate-keyed map — best satisfies the constraints of O(1)
   cell lookup, clean flood-fill traversal, and extensibility to non-rectangular
   topologies in future versions? The answer determines the board module's core
   data type and the interface between board, game logic, and renderer.

2. **[P1]** What terminal rendering approach provides reliable cross-platform
   keyboard input and cursor control on both macOS and Linux without requiring
   a display server, while remaining extensible to the color and animation
   features planned for later versions? The answer determines the renderer
   module's dependency and the rendering abstraction layer.

---

## Non-Goals

- GUI of any kind
- Multiplayer or network play
- Persistent high scores or statistics — future version
- Save and load game state — future version
- Replay — future version
- Automated solver or hints — future version
- Hexagonal or toroidal boards — future version
- Custom color themes — future version
- Windows support
