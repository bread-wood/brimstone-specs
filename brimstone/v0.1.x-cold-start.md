# breadmin-composer v0.1.x — Cold Start

> **Retrospective spec** — describes what was built, not what to build.
> The pipeline has already run for this version.

## Overview

First working release of breadmin-composer. Implements the full pipeline
(spec → init → research → design → scoping → implementation → qa → release)
as a single Python CLI package. Establishes the orchestrator/sub-agent architecture,
subprocess runner, checkpoint persistence, and the `composer` admin command.

---

## What Was Built

| Area | Detail |
|------|--------|
| `composer run` | Unified pipeline runner: `--research`, `--design`, `--impl`, `--all` flags |
| `composer init` | Upload spec + create milestone + seed research issues |
| `composer adopt` | Stub (exits 1, not yet implemented) |
| `composer health` | Preflight health checks |
| `composer cost` | Cost ledger summary |
| Subprocess runner | Isolated env, stream-json output parsing, exit code capture |
| Checkpoint persistence | JSON checkpoint for resume across sessions |
| Orchestrator lock | Prevents concurrent orchestrator sessions |
| CLAUDECODE guard | Prevents nesting orchestrator inside Claude Code |
| yeast-bot auth | `COMPOSER_GH_TOKEN` flows to all worker subprocesses via `GH_TOKEN` |
| Resume recovery | Stale in-progress issues without PRs are unclaimed and re-dispatched |
| Spec seeding | `_upload_spec_to_repo` via GitHub API |
| Milestone gate | `composer run` fails early if milestone doesn't exist |
| Stage gates | Design gate (research complete?), impl gate (HLD exists?) |
| Repo scaffolding | `_scaffold_new_repo` with branch protection |

## Lessons Learned

Key bugs encountered during v0.1.x and what they revealed about spec quality:

| Bug | Root cause | Spec gap |
|-----|-----------|----------|
| Agent output not shown | Stream-json parsing assumed always-present | Output contract not specified |
| Exit codes not captured | Subprocess wrapper didn't record exit code | Interface contract not specified |
| GH_TOKEN not in worker env | `build_subprocess_env` didn't include it | Auth flow across boundaries not specified |
| COMPOSER_ prefix doubled | `AliasChoices` + `env_prefix` interaction | Env var naming contract not specified |
| Stale in-progress not unclaimed | Resume only handled PR-open case | State machine not fully specified |
| Non-init milestone not detected early | No gate before stage dispatch | Failure mode not specified |

**Pattern**: every bug was at a component boundary, not inside a component. Specs must
define behavior at boundaries with the same precision as happy-path functionality.

## Non-Goals (were deferred to v0.2.x)

- Security hardening (credential proxy, API key rotation, sandbox)
- Multi-model backends (Gemini, GPT-4.1, Grok)
- Production CI/server deployment
- Telegram notifications
- Multi-repo orchestration
- Rolling dispatch / agent watchdog
- qa and release stages
